# KP02 蓝图模式（Blueprint Pattern）

## 核心概念

蓝图模式是 SystemVerilog 验证平台中一种非常实用的设计模式。它的核心思想是：**创建一个"蓝图"对象（模板），将其作为工厂，通过调用蓝图对象的 `copy()` 方法来批量生成具有相同初始配置的事务对象**。

**解决的问题**：在验证中，你往往需要大量属性相似的事务（比如相同的地址范围、数据宽度、优先级等）。如果每次都手动设置所有属性，既繁琐又容易出错。蓝图模式让你只需配置一次，之后通过拷贝快速创建同类事务。

---

## 1. 蓝图模式的三个角色

| 角色 | 职责 |
|------|------|
| **蓝图对象（Blueprint）** | 预先配置好所有属性的"模板"对象，本身不参与仿真 |
| **工作对象（Working Copy）** | 通过 `copy()` 从蓝图拷贝而来，参与实际仿真 |
| **copy() 方法** | 负责将蓝图对象的所有属性复制到新对象 |

---

## 2. 基本实现方式

每个类中实现一个 `copy()` 方法，返回一个与自身属性完全相同的新对象：

```systemverilog
function Transaction copy();
    Transaction c = new();
    c.addr  = this.addr;
    c.data  = this.data;
    c.len   = this.len;
    return c;
endfunction
```

---

## 3. 蓝图模式的标准使用流程

```
1. 创建蓝图对象并配置默认属性
2. 在需要生成事务时，调用 blue_print.copy() 获取工作副本
3. 修改工作副本的个别属性（如随机化或覆盖特定字段）
4. 将工作副本发送到验证平台
5. 蓝图对象保持不变，可反复使用
```

---

## 4. 蓝图模式 vs 随机化

蓝图模式和约束随机化并不矛盾，而是**互补**的：
- 蓝图模式设置事务的**整体框架**（哪些字段固定、哪些可变）
- 随机化在蓝图拷贝的基础上**生成变化**（在约束范围内产生不同值）

常见做法：蓝图定义默认值 + randc/rand 约束 → 拷贝 → `randomize()` 产生多种变体。
