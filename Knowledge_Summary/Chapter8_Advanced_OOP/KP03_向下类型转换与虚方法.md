# KP03 向下类型转换（$cast）与虚方法

## 核心概念

在继承体系中，类型转换分为两个方向：
- **向上类型转换**：子类句柄赋给父类句柄，**自动完成**，无需任何操作
- **向下类型转换**：父类句柄赋给子类句柄，**必须使用 `$cast`**，且运行时必须实际指向子类对象

虚方法（`virtual`）则决定了多态调用时到底执行哪个类的方法实现。

---

## 1. 向上类型转换（自动完成）

```systemverilog
BadTxn bad = new("my_bad");
Transaction txn = bad;   // OK，自动完成，bad 仍是 BadTxn 对象
```

父类句柄 `txn` 指向的仍然是 `BadTxn` 对象，但 `txn` 只能访问 `Transaction` 中定义的成员。

---

## 2. 向下类型转换（$cast）

有两种形式：

```systemverilog
// 形式一：函数调用，返回 0（失败）或 1（成功）
result = $cast(target_handle, source_handle);

// 形式二：任务调用，失败时产生运行时错误（推荐，更安全）
$cast(target_handle, source_handle);
```

**关键规则**：`$cast` 不会创建新对象，它只是让编译器"认可"转换。运行时系统会检查**源句柄实际指向的对象类型**是否兼容。

---

## 3. 虚方法（virtual）

### 静态绑定 vs 动态绑定

| 修饰符 | 绑定时机 | 调用规则 |
|:------:|:-------:|---------|
| 无 `virtual` | **编译时**（静态绑定） | 根据句柄类型决定调用哪个方法 |
| `virtual` | **运行时**（动态绑定） | 根据实际对象类型决定调用哪个方法 |

```systemverilog
// 父类中
virtual function void print_type(); ... endfunction   // virtual → 运行时决定
function void display();         ... endfunction       // 非 virtual → 编译时决定
```

### 实际影响

```systemverilog
BadTxn bad = new("bad");
Transaction txn = bad;   // 向上转换
txn.print_type();        // virtual → 调用 BadTxn 的版本
txn.display();           // 非virtual → 调用 Transaction 的版本
```

**验证平台中几乎所有需要被子类覆盖的方法都应声明为 `virtual`**，否则多态将失效。

---

## 4. $cast 的典型使用场景

在验证平台中，驱动器/监视器通常接收父类句柄（`Transaction`），但需要调用子类特有方法。此时需要 `$cast` 还原为子类类型：

```systemverilog
function void drive(Transaction txn);
    ReadWriteTxn rw;
    $cast(rw, txn);              // 向下转换
    rw.rw = 1;                   // 现在可以访问子类特有成员了
endfunction
```
