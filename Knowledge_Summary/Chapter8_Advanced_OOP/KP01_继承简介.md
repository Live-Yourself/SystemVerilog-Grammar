# KP01 继承简介（Inheritance Basics）

## 核心概念

继承是面向对象编程的核心机制之一，它允许你创建一个新类（子类/派生类），基于一个已有的类（父类/基类）进行扩展。子类自动获得父类的所有属性和方法，同时可以添加新的成员或覆盖已有的方法。

**好处**：代码复用、建立类之间的层次关系、统一管理公共逻辑。

---

## 1. 基本继承语法

使用 `extends` 关键字实现继承：

```systemverilog
class 子类名 extends 父类名;
    // 新增或覆盖的成员
endclass
```

- 子类**继承**父类的所有 `public` 和 `protected` 成员
- 子类**无法访问**父类的 `local` 成员
- 子类可以在父类基础上添加新的属性和方法

---

## 2. 构造函数中的 `super.new()`

子类的构造函数中**必须**先调用父类的构造函数 `super.new()`，以确保父类的成员被正确初始化。

- `super.new()` 必须是子类 `new` 函数中的**第一条语句**
- 如果父类构造函数需要参数，子类必须传入对应参数

```systemverilog
function new(string name);
    super.new(name);  // 必须是第一条语句
    // 子类自己的初始化...
endfunction
```

---

## 3. 成员可见性规则

| 访问修饰符 | 类自身 | 子类 | 外部（模块/其他类） |
|:---------:|:-----:|:---:|:----------------:|
| `public`  | Yes   | Yes  | Yes              |
| `protected` | Yes | Yes  | No               |
| `local`   | Yes   | No   | No               |

- 默认情况下，类的成员是 `public` 的
- `local` 成员对子类完全不可见，子类中可以定义**同名**成员而不冲突
- `protected` 是最适合被继承的成员的修饰符

---

## 4. 方法覆盖（Override）

子类可以重新定义父类中的方法（参数签名需一致），这称为**覆盖**。

- 覆盖时不会调用父类的方法，除非显式使用 `super.方法名()`
- 如果要在覆盖的方法中保留父类行为，可以先用 `super.方法名()` 调用父类实现
