# KP05 复制一个对象（深拷贝与浅拷贝）

## 核心概念

在验证平台中，经常需要复制事务对象。比如 Scoreboard 需要保存一份发送的事务，等接收端的事务到了再做对比。如果两个句柄指向同一个对象，任何一方的修改都会影响另一方，导致比对出错。所以必须理解深拷贝和浅拷贝的区别。

---

## 1. 浅拷贝（Shallow Copy）

**只复制句柄**，不复制对象。两个句柄指向同一个对象。

```systemverilog
Transaction t1 = new("t1");
t1.addr = 32'h100;
Transaction t2 = t1;     // 浅拷贝：只是复制了句柄
t2.addr = 32'h200;       // t1.addr 也变成了 0x200！
```

`new()` 只调用了一次，内存中只有一个对象。`t1` 和 `t2` 是同一个对象的两个"遥控器"。

---

## 2. 深拷贝（Deep Copy）

**复制对象本身**，在内存中创建一个全新的对象，逐字段赋值。

```systemverilog
Transaction t1 = new("t1");
t1.addr = 32'h100;
Transaction t2 = new();  // 新建一个对象
t2.addr = t1.addr;       // 逐字段复制
t2.addr = 32'h200;       // t1.addr 不受影响，仍然是 0x100
```

`new()` 调用了两次，内存中有两个独立对象。修改 `t2` 不影响 `t1`。

---

## 3. copy() 方法的两种实现

### 简单深拷贝（字段都是基本类型）

```systemverilog
function Transaction copy();
    Transaction c = new();
    c.addr = this.addr;
    c.data = this.data;
    return c;
endfunction
```

### 包含对象成员时的深拷贝（递归拷贝）

```systemverilog
class Packet;
    Transaction header;   // 成员是另一个对象
    int payload[];

    function Packet copy();
        Packet c = new();
        c.header  = this.header.copy();   // 递归深拷贝内部对象
        c.payload = new[this.payload.size()](this.payload);  // 深拷贝动态数组
        return c;
    endfunction
endclass
```

**如果只写 `c.header = this.header`，那就是浅拷贝**——两个 Packet 共享同一个 header 对象，修改一个会影响另一个。

---

## 4. 什么时候用哪种

| 场景 | 推荐 | 原因 |
|------|:----:|------|
| 只是传递句柄，不需要独立副本 | 浅拷贝 | 高效，不占额外内存 |
| 需要独立修改副本，互不影响 | 深拷贝 | 安全，两个对象完全独立 |
| Scoreboard 保存发送事务用于对比 | 深拷贝 | 发送端后续修改不能影响已保存的副本 |
| 蓝图模式生成工作副本 | 深拷贝 | 工作副本独立修改，蓝图不变 |
