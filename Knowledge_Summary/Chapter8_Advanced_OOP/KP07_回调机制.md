# KP07 回调（Callback Mechanism）

## 核心概念

回调是一种"反向调用"机制：组件 A 在执行过程中，遇到特定事件时自动调用组件 B 事先注册的函数。就像你往公司留了一个电话号码，说"有事打这个电话"——公司（组件 A）在有事时主动打给你（组件 B），而不是你一直打电话问"有事吗"。

**解决的问题**：Monitor 收到事务后，不需要知道 Scoreboard、Coverage Collector 等是谁，只需要按列表依次调用它们注册的函数。实现了组件之间的**解耦**。

---

## 1. 回调的基本组成

| 角色 | 职责 |
|------|------|
| **调用者（Caller）** | 内部维护一个回调函数列表，在特定事件发生时依次调用列表中的函数 |
| **注册者（Register）** | 把自己的函数注册到调用者的列表中 |

---

## 2. 实现方式

### 使用虚方法（书中推荐方式）

```systemverilog
// 调用者内部定义回调用的虚方法
virtual class Callback;
    pure virtual function void pre_trans(Transaction txn);
    pure virtual function void post_trans(Transaction txn);
endclass

class Monitor;
    Callback cbs[$];   // 动态队列，存储所有注册的回调对象

    function void register_cb(Callback cb);
        cbs.push_back(cb);    // 注册
    endfunction

    task run();
        Transaction txn = new();
        // 事件前：调用所有注册的 pre_trans
        foreach (cbs[i]) cbs[i].pre_trans(txn);
        // ... 执行事务 ...
        // 事件后：调用所有注册的 post_trans
        foreach (cbs[i]) cbs[i].post_trans(txn);
    endtask
endclass
```

### 注册者的使用

```systemverilog
class ScoreboardCB extends Callback;
    function void post_trans(Transaction txn);
        $display("Scoreboard 收到事务: %0s", txn.name);
    endfunction
endclass

ScoreboardCB sb = new();
monitor.register_cb(sb);   // 注册
```

---

## 3. 回调 vs 直接调用

| 对比 | 直接调用 | 回调 |
|------|---------|------|
| Monitor 是否知道 Scoreboard | 是，必须直接引用 | 否，完全不知道 |
| 新增 Listener | 修改 Monitor 代码 | 只需新建类并注册 |
| 组件耦合度 | 高（紧耦合） | 低（松耦合） |
