# KP08 参数化的类（Parameterized Class）

## 核心概念

参数化的类就像一个"模板"，你把数据类型、位宽等作为参数传入，编译时自动生成对应的具体类。就像用一个饼干模具，换不同面团就能做出不同口味的饼干——模具（类定义）只有一个，但做出来的饼干（具体类）可以有很多种。

**解决的问题**：当你需要多个结构相同但类型不同的类时，不需要复制粘贴定义多个类，用参数化类一个就够了。

---

## 1. 基本语法

用 `#(parameter)` 定义参数：

```systemverilog
class Stack #(type T = int, int DEPTH = 16);
    T items[$];        // T 在实例化时决定具体类型

    function void push(T val);
        if (items.size() < DEPTH)
            items.push_back(val);
    endfunction
endclass
```

实例化时指定参数：

```systemverilog
Stack #(int)    int_stack;    // T=int, DEPTH=默认16
Stack #(string) str_stack;    // T=string, DEPTH=默认16
Stack #(bit[7:0], 8) byte_stack;  // T=bit[7:0], DEPTH=8
```

---

## 2. 参数类型

| 参数类型 | 示例 | 说明 |
|---------|------|------|
| `type` | `type T` | 可以是任意数据类型（int、string、类名等） |
| `int`/值 | `int WIDTH=8` | 整数常量，用于位宽、深度等 |
| `string` | `string MSG="ok"` | 字符串常量 |

可以同时有多个参数，用逗号分隔，可以有默认值。

---

## 3. 参数化类和继承

子类继承参数化父类时，可以固定参数：

```systemverilog
// 固定 T=int 的子类
class IntStack extends Stack #(int);
    // 不需要再指定 T，已经固定了
endclass

// 保持参数化的子类
class MyStack #(type T) extends Stack #(T);
    // T 继续作为参数传递给父类
endclass
```
