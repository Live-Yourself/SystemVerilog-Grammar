# KP09 静态变量与单例模式（Static Variables & Singleton Pattern）

## 核心概念

### 静态变量

普通成员变量属于"每个对象各自一份"，`static` 成员变量属于"整个类只有一份"。就像公司的公告栏——普通变量是每个人自己的笔记本（各写各的），静态变量是所有人共享的公告栏（改了谁都看得到）。

### 单例模式

单例模式保证一个类在仿真运行期间**只创建一个实例**。全局只有一个对象，所有人共享它。就像一个学校只有一个校长，不管哪个部门要找校长，找到的都是同一个人。

---

## 1. 静态变量（static）

```systemverilog
class Counter;
    static int count = 0;   // 所有对象共享这一个变量

    function new();
        count++;             // 每创建一个对象，count +1
    endfunction
endclass
```

**访问方式**：通过 `类名::变量名` 访问，不需要创建对象：

```systemverilog
$display(Counter::count);   // 直接通过类名访问
```

**注意**：`static` 变量在仿真开始时初始化（只在 `initial` 块之前执行一次），不是每次 `new()` 都初始化。

---

## 2. 静态方法

```systemverilog
class Counter;
    static int count = 0;

    static function int get_count();
        return count;
    endfunction
endclass
```

静态方法也只能通过 `类名::方法名` 调用，不能访问非静态成员（因为没有 `this` 指针）。

---

## 3. 单例模式

核心思路：把构造函数藏起来，只暴露一个静态方法来获取唯一实例：

```systemverilog
class Config;
    protected static Config instance;   // 唯一实例

    // 构造函数设为 protected，外部不能 new
    protected function new();
    endfunction

    // 全局访问点
    static function Config get();
        if (instance == null)
            instance = new();    // 只在第一次调用时创建
        return instance;
    endfunction
endclass
```

使用时：

```systemverilog
Config cfg = Config::get();   // 永远拿到同一个对象
Config cfg2 = Config::get();  // 还是同一个对象
```

---

## 4. 单例模式的典型用途

| 用途 | 说明 |
|------|------|
| 全局配置（Config DB） | 整个验证平台共享一份配置 |
| 事件池（Event Pool） | 集中管理所有事件，避免重复创建 |
| 报告收集器 | 统一收集所有组件的仿真报告 |
