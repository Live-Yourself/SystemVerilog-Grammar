# KP06 抽象类与纯虚方法（Abstract Class & Pure Virtual Method）

## 核心概念

抽象类是一种"不完整"的类，它定义了接口（方法签名），但不提供实现，强制子类去实现。就像老板说"你们每个人都要交周报"（定义了规则），但不规定具体怎么写——每个员工自己决定格式和内容。

---

## 1. 抽象类

用 `virtual class` 关键字定义：

```systemverilog
virtual class Base;
    // ...
endclass
```

**特点**：抽象类**不能直接用 `new()` 创建对象**。它存在的意义就是被继承，让子类来补全缺失的实现。

```systemverilog
Base b = new("b");   // 编译错误！不能实例化抽象类
Child c = new("c");  // OK，子类实现了所有纯虚方法后可以实例化
```

---

## 2. 纯虚方法（Pure Virtual Method）

用 `pure virtual` 关键字声明，**没有方法体**（没有 `endfunction` 之间的代码）：

```systemverilog
virtual class Base;
    pure virtual function void do_work();   // 纯虚方法，没有实现
    pure virtual function string get_type();
endclass
```

**作用**：强制所有子类必须实现这些方法，否则子类也不能被实例化。这保证了每个子类都有完整的实现。

---

## 3. 抽象类的意义

| 好处 | 说明 |
|------|------|
| 统一接口 | 所有子类都有相同的方法签名，调用方不用关心具体类型 |
| 强制实现 | 纯虚方法要求子类必须提供实现，避免遗漏 |
| 代码规范 | 定义了"必须做什么"，但不限制"怎么做" |
| 多态基础 | 父类句柄可以指向任何子类对象，调用纯虚方法时自动分派到子类实现 |

---

## 4. 典型使用模式

验证平台中，抽象类常用于定义**组件基类**：

```systemverilog
virtual class Driver;
    pure virtual function void drive(Transaction txn);
    pure virtual function void reset();
endclass

class ApbDriver extends Driver;
    function void drive(Transaction txn); ... endfunction   // 必须实现
    function void reset(); ... endfunction                   // 必须实现
endclass
```

所有驱动器（APB、AXI、USB...）都继承同一个抽象基类，上层代码通过父类句柄统一调用，不用关心底层协议细节。
