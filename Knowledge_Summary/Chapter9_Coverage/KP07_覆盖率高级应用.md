# KP07 覆盖率高级应用

## 核心概念

本章最后介绍覆盖率在验证平台中的实际应用：参数化覆盖组、覆盖率权重、覆盖组的组织与回调机制。

## 1. 参数化覆盖组

通过 SystemVerilog 的参数机制，使覆盖组更灵活、可复用：

```systemverilog
class AddressMonitor #(int ADDR_WIDTH=8, int NUM_BINS=4);
  rand bit [ADDR_WIDTH-1:0] addr;

  covergroup AddrCov;
    coverpoint addr {
      bins b_range[NUM_BINS] = {[0:(2**ADDR_WIDTH)-1]};  // 动态分箱
    }
  endgroup
endclass

// 实例化不同配置
AddressMonitor #(8, 8)   mon_8bit;    // 8 位地址, 8 个 bin
AddressMonitor #(16, 16) mon_16bit;   // 16 位地址, 16 个 bin
```

参数化覆盖组的好处：一套代码适应不同位宽、不同粒度的需求，提高复用性。

## 2. 覆盖率权重 (weight)

当验证平台有多个 covergroup 时，可以通过 `option.weight` 控制各覆盖组在**总体覆盖率计算**中的权重：

```systemverilog
covergroup ModeCov;
  coverpoint mode { ... }
  option.weight = 10;   // 高权重,更重要
endgroup

covergroup CfgCov;
  coverpoint cfg { ... }
  option.weight = 2;    // 低权重,次要
endgroup
```

总体覆盖率 = 各 covergroup 覆盖率的**加权平均**。权重高的 covergroup 覆盖率不足会显著拉低总体覆盖率，引起更多关注。

## 3. 覆盖组在验证平台中的组织

典型做法是将覆盖率模型**封装在 Transaction 类内部**，随对象创建自动实例化：

```systemverilog
class BusTransaction;
  rand cmd_e cmd;
  rand bit [7:0] addr;

  covergroup TransactionCov;
    coverpoint cmd { ... }
    coverpoint addr { ... }
    cross cmd, addr { ... }
  endgroup

  function new();
    TransactionCov = new();    // 构造时自动创建
  endfunction

  function void sample_cov();
    TransactionCov.sample();   // 封装采样方法
  endfunction
endclass
```

在验证组件（Monitor、Driver）中调用：

```systemverilog
class Monitor;
  BusTransaction txn;

  function new();
    txn = new();               // 覆盖组自动就绪
  endfunction

  task run();
    // 监测总线,收集事务
    txn.sample_cov();          // 一行完成覆盖率采样
  endtask
endclass
```

这种组织方式的优点：覆盖率与数据天然绑定，使用简洁，便于维护。

## 4. 覆盖率最佳实践

1. **覆盖率与数据绑定**：将 covergroup 放在 Transaction 类中，随对象创建
2. **封装采样方法**：提供 `sample_cov()` 方法，避免外部直接调用 `sample()`
3. **合理设置权重**：核心功能覆盖组设置高权重，次要功能设置低权重
4. **使用参数化**：提高覆盖组的复用性
5. **注释和命名规范**：使用 `option.comment` 说明覆盖组用途，bin 命名清晰
6. **交叉覆盖用起来**：单个变量覆盖率 100% 不够，组合场景要关注

## 5. UVM 中的覆盖率

在 UVM 验证平台中，覆盖率通常放在以下位置：
- **Transaction 类**：事务级覆盖率
- **Monitor 组件**：在 `run_phase` 中采样
- **Scoreboard 组件**：比对相关的覆盖率
- **独立的 Coverage Collector**：专用覆盖率收集组件

UVM 还提供了 `uvm_subscriber` 和 analysis port 机制，便于在组件间传递数据并采样覆盖率。
