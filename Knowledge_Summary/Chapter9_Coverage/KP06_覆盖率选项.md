# KP06 覆盖率选项

## 核心概念

覆盖率选项用于精细控制覆盖率收集的行为：每个 bin 需要命中多少次、覆盖率目标、多个实例如何统计等。

## 1. 常用选项一览

| 选项 | 作用域 | 含义 |
|------|--------|------|
| `at_least` | coverpoint | 每个 bin 至少命中 N 次才算覆盖 |
| `auto_bin_max` | coverpoint | 自动分箱的最大数量 |
| `goal` | covergroup | 覆盖率目标百分比，默认 100 |
| `per_instance` | covergroup | 按实例独立统计，而非按类型合并 |
| `comment` | covergroup | 注释说明，显示在覆盖率报告中 |

## 2. option.at_least（最小命中次数）

默认情况下，bin 只需命中 1 次就算覆盖。但验证中可能要求更充分——例如每个 bin 至少命中 5 次，避免"刚好覆盖一次"的偶然性：

```systemverilog
coverpoint cmd {
  bins b0 = {0};
  bins b1 = {1};
  option.at_least = 5;   // 每个 bin 至少命中 5 次
}
```

如果某 bin 命中次数 < at_least，在覆盖率报告中该 bin 显示为**未覆盖**，即使值曾经出现过。

## 3. option.goal（覆盖率目标）

`goal` 设置覆盖组的覆盖率目标百分比，默认 100。覆盖率报告中会标记"是否达到目标"：

```systemverilog
covergroup MyCov;
  coverpoint addr { ... }
  option.goal = 80;   // 目标 80%,不是必须 100%
endgroup
```

这在阶段性验收中很实用——例如 Phase1 目标 80%，Phase2 目标 100%。

## 4. option.per_instance（实例级统计）

**默认行为**：同一个 covergroup 类型的所有实例覆盖率**合并统计**，报告一个总体覆盖率。

**per_instance=1**：每个实例**独立统计**，报告各实例的覆盖率：

```systemverilog
covergroup InstanceCov;
  coverpoint cmd { bins b[4] = {[0:3]}; }
  option.per_instance = 1;   // 各实例独立统计
endgroup

ConfigPacket pkt1 = new();
ConfigPacket pkt2 = new();
// pkt1.InstanceCov 和 pkt2.InstanceCov 独立计算覆盖率
```

典型场景：多个驱动器/监视器各自统计覆盖率，便于定位哪个组件验证不充分。

## 5. option.auto_bin_max（自动分箱上限）

已在 KP03 讲过，用于控制自动分箱的最大数量：

```systemverilog
coverpoint addr {
  option.auto_bin_max = 16;   // 最多 16 个自动 bin
}
```

## 6. type_option vs instance_option

| 类型 | 作用域 | 设置方式 |
|------|--------|----------|
| `option` | 可用于 covergroup / coverpoint | 定义时设置 |
| `type_option` | covergroup 类型级 | 所有实例共享 |
| `instance_option` | covergroup 实例级 | 各实例独立设置 |

示例：

```systemverilog
covergroup MyCov;
  coverpoint addr { ... }
  type_option.comment = "Type-level comment";      // 类型级
  option.per_instance = 1;                         // 可在定义时设置
endgroup

MyCov cov1 = new();
cov1.instance_option.comment = "Instance 1";       // 实例级设置
```
