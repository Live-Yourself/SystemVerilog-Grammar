## 知识点11: 对象的比较

### 核心概念

对象的比较是验证平台中非常关键的操作,尤其在Scoreboard中比对期望值和实际值时不可或缺。书中明确区分了两种比较方式:**句柄比较**和**内容比较**。默认的`==`和`!=`操作符比较的是句柄(即两个句柄是否指向同一个对象),而不是对象的内容。要比较对象的内容,需要自定义比较方法。

### 句柄比较(== / !=)

默认情况下,`==`和`!=`比较的是**句柄地址**,即判断两个句柄是否指向**同一个对象**:

```systemverilog
Transaction tr1 = new(32'h1000, 32'hABCD, 1);
Transaction tr2 = tr1;           // 句柄赋值,指向同一对象
Transaction tr3 = new(32'h1000, 32'hABCD, 1);  // 内容相同但不同对象

if (tr1 == tr2)  // true  -- 同一个对象
  $display("tr1和tr2是同一个对象");

if (tr1 == tr3)  // false -- 内容相同但不同对象!
  $display("这句不会执行");

if (tr1 != tr3)  // true  -- 句柄不同
  $display("tr1和tr3不是同一个对象");
```

即使`tr1`和`tr3`的所有成员变量值完全相同,`tr1 == tr3`仍然返回`false`,因为它们是两个独立的对象,内存地址不同。

### 句柄比较的本质

```
tr1 ──→ [对象A: addr=0x1000, data=0xABCD]     地址: 0x1000
tr2 ──→ [对象A: addr=0x1000, data=0xABCD]     地址: 0x1000  (同tr1)
tr3 ──→ [对象B: addr=0x1000, data=0xABCD]     地址: 0x2000

tr1 == tr2  → true  (地址相同: 0x1000 == 0x1000)
tr1 == tr3  → false (地址不同: 0x1000 != 0x2000)
```

句柄比较只是指针地址的比较,不关心对象内容。

### 内容比较 -- 自定义compare()方法

要比较对象的内容,需要在类中定义一个`compare()`方法:

```systemverilog
class Transaction;
  bit [31:0] addr;
  bit [31:0] data;
  bit        write;

  function bit compare(Transaction tr);
    if (this.addr == tr.addr &&
        this.data == tr.data &&
        this.write == tr.write)
      return 1;   // 所有字段相等
    else
      return 0;   // 存在差异
  endfunction
endclass

Transaction tr1 = new(32'h1000, 32'hABCD, 1);
Transaction tr2 = new(32'h1000, 32'hABCD, 1);
Transaction tr3 = new(32'h2000, 32'hABCD, 1);

tr1.compare(tr2);  // 返回1,内容相同
tr1.compare(tr3);  // 返回0,addr不同
```

### compare()方法的标准实现模式

书中推荐`compare()`方法返回`bit`类型: 1表示相等, 0表示不等:

```systemverilog
class Transaction;
  bit [31:0] addr;
  bit [31:0] data;
  bit        write;

  function bit compare(Transaction tr);
    compare = 1;  // 默认相等
    if (this.addr != tr.addr) begin
      $display("  [FAIL] addr不匹配: 0x%08h vs 0x%08h", this.addr, tr.addr);
      compare = 0;
    end
    if (this.data != tr.data) begin
      $display("  [FAIL] data不匹配: 0x%08h vs 0x%08h", this.data, tr.data);
      compare = 0;
    end
    if (this.write != tr.write) begin
      $display("  [FAIL] write不匹配: %0d vs %0d", this.write, tr.write);
      compare = 0;
    end
  endfunction
endclass
```

带详细不匹配报告的`compare()`方法在Scoreboard中非常实用,能快速定位哪个字段不一致。

### 比较含嵌套对象的compare()

当类包含嵌套对象时,`compare()`需要递归比较嵌套对象的内容:

```systemverilog
class Header;
  bit [7:0] src;
  bit [7:0] dst;

  function bit compare(Header h);
    if (this.src == h.src && this.dst == h.dst)
      return 1;
    else
      return 0;
  endfunction
endclass

class Packet;
  bit [31:0] addr;
  Header     hdr;

  function bit compare(Packet p);
    if (this.addr != p.addr)
      return 0;
    if (this.hdr == null || p.hdr == null)
      return 0;
    return this.hdr.compare(p.hdr);  // 递归比较嵌套对象
  endfunction
endclass
```

### null句柄的比较

使用`==`和`!=`可以判断句柄是否为`null`:

```systemverilog
Transaction tr;

if (tr == null)     // true -- 句柄为null
  $display("tr为null,未指向任何对象");

if (tr != null)     // false
  $display("这句不会执行");

// 创建对象后
tr = new();
if (tr != null)     // true
  $display("tr已指向一个对象");
```

两个`null`句柄比较也返回相等:

```systemverilog
Transaction tr1, tr2;
if (tr1 == tr2)     // true -- 两个null相等
  $display("两个null句柄相等");
```

### 句柄比较 vs 内容比较 总结

| 特性 | 句柄比较 (`==`) | 内容比较 (`compare()`) |
|------|----------------|---------------------|
| **比较对象** | 句柄地址 | 成员变量值 |
| **语法** | `tr1 == tr2` | `tr1.compare(tr2)` |
| **系统内置** | 是,默认行为 | 否,需自定义 |
| **内容相同但不同对象** | false | true |
| **同一对象** | true | true |
| **典型用途** | 判断是否同一对象 | Scoreboard数据比对 |

### 在Scoreboard中的应用

Scoreboard是验证平台中进行对象比较最频繁的地方:

```systemverilog
class Scoreboard;
  function void check(Transaction expected, Transaction actual);
    if (expected.compare(actual))
      $display("[PASS] 期望值与实际值匹配");
    else
      $display("[FAIL] 期望值与实际值不匹配");
  endfunction
endclass
```

`expected`来自参考模型,`actual`来自DUT的输出 Monitor。Scoreboard通过`compare()`判断DUT行为是否符合预期。

### 关键要点

| 要点 | 说明 |
|------|------|
| **默认`==`** | 比较句柄地址,不比较内容 |
| **内容相同≠同一对象** | 成员值一样但内存地址不同的两个对象,`==`返回false |
| **null比较** | `null == null`为true, `tr == null`判断句柄是否为空 |
| **compare()方法** | 自定义方法,逐字段比较成员值 |
| **返回值** | compare()通常返回`bit`: 1相等, 0不等 |
| **嵌套对象** | compare()需递归比较嵌套对象的内容 |
| **Scoreboard** | 验证平台中compare()最核心的应用场景 |

### 典型应用场景

- **Scoreboard比对**: 比较期望事务和实际事务是否一致
- **事务匹配**: 在响应队列中查找匹配的事务
- **配置检查**: 验证当前配置是否与期望配置一致
- **去重判断**: 检查新事务是否与已处理的事务内容重复
