## 知识点6: 静态变量与静态方法(static)

### 核心概念

使用`static`关键字声明的成员属于**类级别**而非对象级别。普通成员变量在每个对象中都有独立副本,而静态变量只有一份,被该类的所有对象共享。书中强调:静态成员在类被加载时初始化,而非在对象创建时初始化。

### 静态变量(static variable)

```systemverilog
class Transaction;
  bit [31:0] addr;        // 实例变量: 每个对象独立
  static int count = 0;   // 静态变量: 所有对象共享同一份

  function new();
    count = count + 1;    // 每创建一个对象,count递增
  endfunction
endclass

Transaction t1 = new();  // count = 1
Transaction t2 = new();  // count = 2
Transaction t3 = new();  // count = 3
// t1.count == t2.count == t3.count == 3
```

### 静态变量的两种访问方式

```systemverilog
// 方式1: 通过类名访问(推荐)
$display(Transaction::count);  // 不需要创建对象

// 方式2: 通过对象句柄访问
Transaction t = new();
$display(t.count);  // 也可以,但不推荐(容易误解为实例变量)
```

**推荐**: 始终用`类名::变量名`的方式访问静态变量,这样语义更清晰,表明它是类级别的。

### 静态方法(static method)

静态方法同样属于类级别,不能访问实例成员(因为没有`this`指针):

```systemverilog
class Transaction;
  static int count = 0;

  // 静态方法: 只能访问静态成员
  static function int get_count();
    return count;  // 可以访问静态变量
    // return addr; // 错误! 不能访问实例变量
  endfunction
endclass

// 通过类名调用静态方法
$display(Transaction::get_count());
```

### 静态方法 vs 实例方法

| 特性 | 静态方法 | 实例方法 |
|------|---------|---------|
| **声明** | `static function ...` | `function ...` |
| **访问实例变量** | 不可以 | 可以 |
| **访问静态变量** | 可以 | 可以 |
| **调用方式** | `类名::方法名()` | `句柄.方法名()` |
| **使用this** | 不可以 | 可以 |
| **典型用途** | 工具方法、全局计数 | 对象行为 |

### 静态变量实现对象计数器

这是静态变量最经典的应用,自动统计创建了多少个对象:

```systemverilog
class Packet;
  static int total_count = 0;
  int id;

  function new();
    total_count = total_count + 1;
    id = total_count;
  endfunction

  static function int get_total();
    return total_count;
  endfunction
endclass

Packet p1 = new();  // id=1, total_count=1
Packet p2 = new();  // id=2, total_count=2
$display("已创建 %0d 个Packet", Packet::get_total());  // 输出: 2
```

### 静态变量用于全局配置

验证平台中常用静态变量存储全局配置参数:

```systemverilog
class Config;
  static bit [31:0] base_addr = 32'h0000_0000;
  static int         timeout   = 1000;
  static bit         verbose   = 0;

  static function void set_base_addr(bit [31:0] addr);
    base_addr = addr;
  endfunction
endclass

// 在任何地方都可以访问全局配置
Config::base_addr = 32'h8000_0000;
Config::verbose   = 1;
```

### 静态变量的初始化

- 静态变量在仿真开始时(类加载时)执行一次初始化
- 如果没有显式初始化,默认值为0(对于整型)或null(对于对象句柄)
- 静态变量的初始化不会在每次`new()`时重复执行

### 关键要点

- 静态成员属于**类级别**,被所有对象共享,只有一份
- 用`static`关键字声明静态变量和静态方法
- 静态变量通过`类名::变量名`或`句柄.变量名`访问
- 静态方法通过`类名::方法名()`调用
- 静态方法**不能**访问实例成员(没有`this`)
- 典型用途: 对象计数、全局配置、工具方法

### 典型应用场景

- **对象计数器**: 自动统计创建了多少个Transaction/Packet
- **唯一ID生成**: 为每个对象分配不重复的ID
- **全局配置**: 存储仿真参数、超时时间、调试开关
- **工具方法**: 提供与具体对象无关的通用操作函数
