## 知识点5: 对象的赋值与复制

### 核心概念

书中明确区分了两种操作:**句柄赋值**(Handle Assignment)和**对象复制**(Object Copy)。这是SystemVerilog面向对象编程中最容易混淆的概念之一。

- **句柄赋值**: `tr2 = tr1` -- 复制的是地址引用,`tr1`和`tr2`指向同一个对象
- **对象复制**: 创建一个全新的对象,内容和原对象相同,但内存独立

### 句柄赋值 vs 对象复制

```
句柄赋值:                对象复制:
tr1 ──→ [对象X]         tr1 ──→ [对象X]
                ↓                      tr2 ──→ [对象X']  (独立副本)
tr2 ──→ [对象X]
(tr1和tr2共享同一对象)   (tr1和tr2各指向独立对象)
```

### 句柄赋值的本质

```systemverilog
Transaction tr1 = new(32'h1000, 32'hABCD, 1);
Transaction tr2;

tr2 = tr1;  // 句柄赋值: tr2复制tr1的地址

// tr1和tr2指向同一对象
// 修改tr2.addr, tr1.addr也跟着变!
tr2.addr = 32'h2000;
// tr1.addr == 32'h2000  ← 因为是同一个对象
```

### 浅复制(Shallow Copy)

浅复制只复制对象的第一层成员变量。对于非对象类型的成员(如`bit`、`int`)是真正的值复制;但对于对象类型的成员(如嵌套的类实例),复制的仍然是句柄引用。

```systemverilog
class Header;
  bit [7:0] src;
  bit [7:0] dst;
endclass

class Packet;
  bit [31:0] addr;  // 非对象成员 → 值复制
  Header     hdr;   // 对象成员   → 句柄复制!
endclass

Packet p1 = new();
p1.addr = 32'h1000;
p1.hdr = new();
p1.hdr.src = 8'h10;

Packet p2;
p2 = p1;  // 浅复制

// 非对象成员:独立
p2.addr = 32'h2000;
// p1.addr 仍为 0x1000  ← 独立的

// 对象成员:共享!
p2.hdr.src = 8'hFF;
// p1.hdr.src 也变成 0xFF  ← 因为p1.hdr和p2.hdr指向同一个Header!
```

**浅复制的限制**: 嵌套对象仍然被共享,修改副本的嵌套成员会影响原对象。

### 深复制(Deep Copy)

深复制创建一个完全独立的对象副本,包括所有嵌套对象也重新创建:

```systemverilog
class Packet;
  bit [31:0] addr;
  Header     hdr;

  // 深复制方法: 复制所有层级
  function Packet copy();
    Packet p = new();
    p.addr = this.addr;          // 值复制
    p.hdr  = new();              // 创建新的嵌套对象
    p.hdr.src = this.hdr.src;    // 复制嵌套对象的值
    p.hdr.dst = this.hdr.dst;
    return p;
  endfunction
endclass
```

深复制后,原对象和副本完全独立,修改任一方不影响另一方。

### copy()方法的标准实现模式

书中推荐在类中定义一个`copy()`方法,这是验证平台中的常见模式(UVM中`uvm_object`也提供`copy()`方法):

```systemverilog
class Transaction;
  bit [31:0] addr;
  bit [31:0] data;
  bit        write;

  function Transaction copy();
    Transaction t = new();
    t.addr  = this.addr;
    t.data  = this.data;
    t.write = this.write;
    return t;
  endfunction
endclass

// 使用
Transaction tr1 = new(...);
Transaction tr2 = tr1.copy();  // tr2是tr1的独立副本

tr2.addr = 32'h9999;
// tr1.addr不受影响,因为tr2是完全独立的对象
```

### 带参构造实现一次性复制

另一种方式是通过带参构造函数在创建对象时完成复制:

```systemverilog
class Transaction;
  bit [31:0] addr;
  bit [31:0] data;
  bit        write;

  function new(bit [31:0] a, bit [31:0] d, bit w);
    addr = a; data = d; write = w;
  endfunction

  // 用另一个Transaction来初始化
  function new(Transaction t);
    addr  = t.addr;
    data  = t.data;
    write = t.write;
  endfunction
endclass

Transaction tr1 = new(32'h1000, 32'hABCD, 1);
Transaction tr2 = new(tr1);  // 用tr1的值构造tr2
```

### 关键要点

- `tr2 = tr1`是**句柄赋值**,两个句柄共享同一对象
- **浅复制**只复制第一层成员,嵌套对象仍是共享的
- **深复制**递归复制所有层级,创建完全独立的对象
- 标准做法是定义`copy()`方法返回独立副本
- 深复制需要在`copy()`中对每个嵌套对象调用`new()`

### 典型应用场景

- **Driver发送事务**: 从Generator获取事务后,用`copy()`创建独立副本发送,避免修改原始事务
- **Scoreboard比对**: 保存期望事务的副本用于后续比对
- **并行处理**: 多个组件需要独立的事务副本同时操作
- **随机化保留**: 保留原始随机值,在副本上做修改
