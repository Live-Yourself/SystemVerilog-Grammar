## 知识点9: 类的作用域

### 核心概念

SystemVerilog中的作用域(Scope)决定了变量和标识符的可见性和生命周期。书中将作用域分为几个层次:**$unit作用域**(编译单元级别)、**类作用域**(类级别)、**方法作用域**(方法内部的局部变量)。理解作用域层次有助于避免命名冲突,正确判断变量在何处可见、何时存在。

### 作用域层次结构

```
$unit作用域 (最外层)
├── 类A的作用域
│   ├── 成员变量 (类级别)
│   ├── 方法1的作用域
│   │   └── 局部变量 (方法级别,最内层)
│   └── 方法2的作用域
│       └── 局部变量
├── 类B的作用域
│   └── ...
└── 模块/程序块作用域
```

**查找规则**: 当访问一个标识符时,编译器从**最内层**作用域开始查找,逐层向外。如果内层定义了同名变量,就会**遮蔽**(shadow)外层的同名变量。

### $unit作用域

`$unit`是编译单元级别的作用域。在类、模块、程序块外部声明的变量和类型属于`$unit`作用域,可以在同一编译单元内的任何地方访问:

```systemverilog
// $unit作用域: 在类外部声明
typedef enum { READ, WRITE } op_t;   // $unit中的类型定义
int global_counter = 0;               // $unit中的全局变量

class Transaction;
  op_t op;  // 可以使用$unit中定义的类型
  // ...
endclass
```

### 类作用域 -- 成员变量的可见性

在类内部声明的成员变量属于类作用域,在该类的**所有实例方法**中都可以访问:

```systemverilog
class Transaction;
  bit [31:0] addr;   // 类作用域
  bit [31:0] data;   // 类作用域

  function void display();
    // addr和data在类作用域中,此处可直接访问
    $display("addr=0x%08h, data=0x%08h", addr, data);
  endfunction

  function void set(bit [31:0] a);
    // addr在类作用域中,此处也可访问
    addr = a;  // 直接使用成员变量
  endfunction
endclass
```

成员变量不局限于某个方法,而是属于整个类,所有方法共享。

### 方法作用域 -- 局部变量

在方法内部声明的变量是局部变量,只在**该方法内部**可见,方法执行完毕后就不存在了:

```systemverilog
function void calculate();
  bit [31:0] temp;     // 局部变量,仅在此方法内可见
  int i;               // 局部变量
  temp = addr + data;
  $display("temp=0x%08h", temp);
endfunction

function void display();
  // temp在此不可见! 它是calculate()的局部变量
  $display("addr=0x%08h", addr);  // addr仍可见(类作用域)
endfunction
```

局部变量和参数只在声明它的方法内部有效,不同方法的局部变量互相独立。

### 局部变量遮蔽成员变量(Shadowing)

当方法的局部变量(或参数)与类的成员变量同名时,局部变量会**遮蔽**成员变量:

```systemverilog
class Transaction;
  bit [31:0] addr;

  function void set(bit [31:0] addr);  // 参数addr遮蔽成员addr
    // 不带this的addr → 局部参数
    // this.addr     → 成员变量
    addr = 32'hFFFF;       // 修改的是参数,不是成员变量!
    this.addr = 32'h1000;  // 修改的是成员变量
  endfunction
endclass
```

遮蔽规则: **内层作用域的同名变量优先**。不加`this`时访问的是局部变量,加`this.`才能访问被遮蔽的成员变量。

### 不同方法的局部变量互不影响

每个方法有自己的局部作用域,同名局部变量在不同方法中完全独立:

```systemverilog
class Transaction;
  function void method_a();
    int count = 0;      // method_a的局部count
    count = count + 1;
    $display("method_a: count=%0d", count);  // 输出1
  endfunction

  function void method_b();
    int count = 100;    // method_b的局部count,与method_a无关
    count = count + 1;
    $display("method_b: count=%0d", count);  // 输出101
  endfunction
endclass
```

### begin/end块中的局部作用域

`begin/end`块可以创建嵌套的局部作用域,块内声明的变量在块外不可见:

```systemverilog
function void test();
  int x = 10;       // 外层局部变量
  begin
    int x = 20;     // 内层局部变量,遮蔽外层的x
    $display("内层 x=%0d", x);   // 输出20
  end
  $display("外层 x=%0d", x);     // 输出10,不受内层影响
endfunction
```

### 类作用域解析操作符(::)

`::`操作符用于显式指定标识符所属的作用域:

```systemverilog
class Transaction;
  static int count = 0;
  // ...
endclass

// 用::显式指定从类作用域访问静态成员
int n = Transaction::count;

// $unit作用域的全局变量也可以用$unit::访问
$unit::global_counter = 5;
```

### 类内嵌套声明的类作用域

SystemVerilog允许在类内部定义新的类型(typedef),这些类型属于该类的作用域:

```systemverilog
class Packet;
  typedef enum { GOOD, BAD, ERROR } status_t;  // 类作用域内的类型

  status_t status;

  function void set_good();
    status = GOOD;            // 直接使用,同作用域
  endfunction
endclass

// 外部访问需要用类名限定
Packet::status_t s = Packet::GOOD;
```

### 变量查找顺序总结

当在代码中引用一个标识符时,编译器按以下顺序查找:

1. **当前块作用域**(begin/end内的局部变量)
2. **方法作用域**(方法参数和局部变量)
3. **类作用域**(类的成员变量)
4. **$unit作用域**(编译单元级别的声明)

找到第一个匹配的标识符即停止查找。内层同名变量遮蔽外层同名变量。

### 关键要点

| 要点 | 说明 |
|------|------|
| **$unit作用域** | 编译单元级别,类/模块外部声明的标识符 |
| **类作用域** | 类内部声明的成员变量,所有方法共享 |
| **方法作用域** | 方法内部的局部变量,仅该方法可见 |
| **遮蔽规则** | 内层同名变量遮蔽外层同名变量 |
| **查找顺序** | 块 → 方法 → 类 → $unit,由内向外 |
| **::操作符** | 显式指定作用域,如`类名::静态成员` |
| **局部变量独立** | 不同方法的同名局部变量互不影响 |

### 典型应用场景

- **避免命名冲突**: 理解遮蔽规则,避免因同名导致的意外行为
- **合理使用this**: 遮蔽发生时,用`this.`显式访问成员变量
- **$unit类型定义**: 将公共的typedef、常量放在$unit作用域供多个类共享
- **作用域限定**: 用`::`操作符解决跨作用域的访问需求
- **代码可读性**: 合理规划变量命名,避免不必要的遮蔽
