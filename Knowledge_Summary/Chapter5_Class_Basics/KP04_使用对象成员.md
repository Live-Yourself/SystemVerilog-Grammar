## 知识点4: 使用对象成员

### 核心概念

通过**点号访问符**(.)可以访问对象的成员变量和调用成员方法。这是与对象交互的基本方式。书中强调:只要拥有了对象的句柄,就可以自由地读写该对象的成员变量和调用其方法。

### 访问成员变量

使用`句柄.变量名`来读写对象的成员变量:

```systemverilog
Transaction tr = new(...);

// 读取成员变量
bit [31:0] current_addr = tr.addr;

// 写入成员变量
tr.addr = 32'h1000_0000;
tr.data = 32'hDEAD_BEEF;
tr.write = 1;
```

### 调用成员方法

使用`句柄.方法名(参数)`来调用对象的成员方法:

```systemverilog
Transaction tr = new(32'h1000, 32'hABCD, 1);

tr.display();           // 调用无参方法
tr.set_addr(32'h2000);  // 调用有参方法
```

### 在方法中访问当前对象的成员

在成员方法内部,可以直接使用成员名,不需要加句柄前缀:

```systemverilog
class Transaction;
  bit [31:0] addr;

  function void display();
    $display("addr = 0x%08h", addr);  // 直接用成员名,不需要this.addr
  endfunction
endclass
```

### 方法修改成员变量

成员方法可以直接修改对象自身的成员变量,修改对持有该对象句柄的所有调用者可见:

```systemverilog
class Transaction;
  bit [31:0] addr;
  bit [31:0] data;

  function void set(bit [31:0] a, bit [31:0] d);
    addr = a;  // 修改当前对象的成员
    data = d;
  endfunction
endclass

Transaction tr = new(...);
tr.set(32'h1000, 32'hABCD);
// tr.addr和tr.data已被修改
```

### 方法访问其他成员方法

成员方法内部可以调用同一对象的其他成员方法:

```systemverilog
class Transaction;
  function void init(bit [31:0] a, bit [31:0] d);
    addr = a;
    data = d;
  endfunction

  function void init_and_display(bit [31:0] a, bit [31:0] d);
    init(a, d);   // 调用同一对象的另一个方法
    display();    // 再调用display
  endfunction
endclass
```

### 嵌套对象访问

当类的成员变量本身也是对象时,使用连续的点号访问:

```systemverilog
class Header;
  bit [7:0] src_addr;
  bit [7:0] dst_addr;
endclass

class Packet;
  Header hdr;          // Packet包含一个Header对象
  bit [15:0] length;
endclass

Packet pkt = new();
pkt.hdr = new();             // 创建嵌套对象
pkt.hdr.src_addr = 8'h10;    // 通过连续点号访问嵌套成员
pkt.hdr.dst_addr = 8'h20;
pkt.length = 16'h0040;
```

### null句柄访问的后果

对值为`null`的句柄使用点号访问会导致运行时错误(空指针解引用):

```systemverilog
Transaction tr;       // tr = null
tr.display();         // 运行时错误! tr未指向任何对象
tr.addr = 32'h1000;   // 同样是运行时错误!
```

**最佳实践**: 使用前先检查句柄是否为null:

```systemverilog
if (tr != null) begin
  tr.display();
  tr.addr = 32'h1000;
end
```

### 关键要点

- 使用**点号(.)**访问对象的成员变量和成员方法
- 成员方法内部直接用成员名访问,不需要句柄前缀
- 方法中对成员变量的修改对所有持有该句柄的调用者可见
- 嵌套对象通过连续点号访问(如`pkt.hdr.src_addr`)
- 对`null`句柄使用点号会导致运行时错误
- 访问前应检查句柄是否为`null`

### 典型应用场景

- **事务配置**: 创建事务后通过点号设置地址、数据等字段
- **组件交互**: Driver通过点号调用接口方法发送事务
- **状态查询**: 通过点号读取Monitor采集到的信号值
- **链式操作**: 对象方法返回句柄时,可连续点号调用
