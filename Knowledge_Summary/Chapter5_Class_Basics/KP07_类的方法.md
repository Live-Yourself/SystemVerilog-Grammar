## 知识点7: 类的方法

### 核心概念

类的方法是定义在类内部的`function`(函数)和`task`(任务)。书中将方法分为两类:函数用于执行计算并返回值,不消耗仿真时间;任务可以包含时序控制语句(如`#delay`、`@(posedge clk)`),会消耗仿真时间。

### function vs task 的区别

| 特性 | function(函数) | task(任务) |
|------|---------------|-----------|
| **返回值** | 可以有返回值 | 不能有返回值 |
| **时序控制** | 不允许(不能有`#`、`@`、`wait`) | 允许 |
| **消耗仿真时间** | 不消耗 | 可以消耗 |
| **调用方式** | 可在表达式中使用 | 只能作为语句调用 |
| **典型用途** | 计算、查询、转换 | 驱动、等待、通信 |

### function的定义与使用

```systemverilog
class Transaction;
  bit [31:0] addr;
  bit [31:0] data;

  // 无返回值的函数
  function void set_addr(bit [31:0] a);
    addr = a;
  endfunction

  // 有返回值的函数
  function bit [31:0] get_addr();
    return addr;
  endfunction

  // 带多个参数的函数
  function void set(bit [31:0] a, bit [31:0] d);
    addr = a;
    data = d;
  endfunction
endclass

Transaction tr = new();
tr.set_addr(32'h1000);              // 调用无返回值函数
bit [31:0] a = tr.get_addr();       // 调用有返回值函数,可在表达式中使用
tr.set(32'h2000, 32'hABCD);         // 调用多参数函数
```

### task的定义与使用

```systemverilog
class Driver;
  virtual clk_if vif;  // 虚拟接口

  // task可以包含时序控制
  task send(bit [31:0] addr, bit [31:0] data);
    @(posedge vif.clk);      // 等待时钟上升沿
    vif.addr  <= addr;
    vif.data  <= data;
    vif.valid <= 1;
    @(posedge vif.clk);
    vif.valid <= 0;
  endtask

  // task可以调用其他task
  task send_transaction(Transaction tr);
    send(tr.addr, tr.data);
  endtask
endclass
```

### 方法参数的传递方式

书中介绍了三种参数方向:

```systemverilog
function void example(
  input  int a,      // 输入: 调用时传入,方法内部可读
  output int b,      // 输出: 方法内部赋值,返回给调用者
  inout  int c       // 双向: 传入并可修改后返回
);
  b = a + c;         // 修改output参数
  c = c * 2;         // 修改inout参数
endfunction
```

| 方向 | 传入时 | 返回时 | 用途 |
|------|--------|--------|------|
| `input` | 复制传入 | 不返回 | 给方法提供数据(默认) |
| `output` | 不传入 | 复制返回 | 方法向调用者返回数据 |
| `inout` | 复制传入 | 复制返回 | 双向传递 |

### 参数的默认值和默认方向

如果不指定参数方向,默认为`input`。可以为参数设置默认值:

```systemverilog
function void display(string prefix = "INFO");
  $display("[%0s] addr=0x%08h", prefix, addr);
endfunction

tr.display();              // 使用默认值 "INFO"
tr.display("DEBUG");       // 传入自定义值
```

### void返回类型

不返回值的函数必须声明为`void`:

```systemverilog
function void print_info();  // void表示无返回值
  $display("addr=0x%08h", addr);
endfunction
```

### 方法中访问成员

在方法内部可以直接访问同对象的成员变量和其他方法(知识点4已讲):

```systemverilog
function void reset();
  addr  = 0;     // 直接访问成员变量
  data  = 0;
  write = 0;
  $display("reset done");  // 可以调用系统函数
endfunction
```

### 方法的可见性(简要)

SystemVerilog使用`local`和`protected`关键字控制方法的访问:

```systemverilog
class Transaction;
  bit [31:0] addr;

  // 公有方法(默认): 外部可调用
  function void set_addr(bit [31:0] a);
    addr = a;
  endfunction

  // 局部方法: 仅类内部可调用
  local function void validate();
    // 内部校验逻辑,外部不可调用
  endfunction

  // 保护方法: 类及其子类可调用
  protected function void internal_check();
    // 子类也可访问
  endfunction
endclass
```

### 关键要点

- **function**不消耗仿真时间,可以有返回值,可在表达式中使用
- **task**可以包含时序控制(`#`、`@`、`wait`),消耗仿真时间,无返回值
- 参数方向: `input`(默认)、`output`、`inout`
- 不返回值的函数必须用`void`声明
- 方法内部可直接访问同对象的所有成员

### 典型应用场景

- **function**: 数据计算(`calc_checksum`)、状态查询(`is_empty`)、格式转换(`to_string`)
- **task**: 信号驱动(`drive_signal`)、时钟同步操作(`wait_for_done`)、事务发送(`send_transaction`)
