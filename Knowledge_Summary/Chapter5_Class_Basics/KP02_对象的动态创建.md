## 知识点2: 对象的动态创建

### 构造函数的概念

| 特性 | 说明 |
|------|------|
| **函数名** | new() |
| **作用** | 分配内存并初始化对象 |
| **返回值** | 返回对象句柄(自动) |
| **调用时机** | 创建对象时自动调用 |
| **是否必需** | 不定义时使用默认构造函数 |

### 默认构造函数

SystemVerilog为每个类提供默认的无参构造函数:

```systemverilog
class Transaction;
  bit [31:0] addr;
  bit [31:0] data;
  // 如果没有显式定义new(),编译器自动生成:
  // function new();
  //   // 空实现,成员变量使用默认值
  // endfunction
endclass
```

### 自定义构造函数

```systemverilog
class Transaction;
  bit [31:0] addr;
  bit [31:0] data;
  bit        write;
  
  // 自定义构造函数 - 无参数
  function new();
    addr  = 32'h0000_0000;
    data  = 32'h0000_0000;
    write = 0;
  endfunction
  
  // 自定义构造函数 - 带参数
  function new(input bit [31:0] a, input bit [31:0] d, input bit w);
    addr  = a;
    data  = d;
    write = w;
  endfunction
endclass

// 使用方式
Transaction trans1 = new();                      // 调用无参构造
Transaction trans2 = new(32'h1000, 32'hABCD, 1); // 调用有参构造
```

### 构造函数的重要特性

| 特性 | 说明 |
|------|------|
| **自动调用** | new()在对象创建时自动调用,不能手动调用 |
| **无返回类型** | 构造函数不声明返回类型,但自动返回对象句柄 |
| **可重载** | 通过不同参数列表实现多个new()方法 |
| **初始化顺序** | 成员变量初始化 → 构造函数执行 |
| **this关键字** | 构造函数中可使用this指向当前对象 |

### 对象创建的完整流程

```
步骤1: 声明句柄
Transaction trans;  // 句柄trans = null

步骤2: 分配内存
new() → 在堆区分配内存空间

步骤3: 初始化成员变量
所有成员变量设置为默认值(0, null等)

步骤4: 执行构造函数
执行new()函数体内的代码

步骤5: 返回句柄
将对象地址赋给句柄trans
```

### 构造函数重载示例

```systemverilog
class Packet;
  bit [7:0] data[];
  bit       valid;
  
  // 构造函数1: 无参数
  function new();
    data  = new[8];  // 分配8字节数组
    valid = 0;
  endfunction
  
  // 构造函数2: 指定数据大小
  function new(int size);
    data  = new[size];
    valid = 0;
  endfunction
  
  // 构造函数3: 指定数据和有效性
  function new(input bit [7:0] d[], input bit v);
    data  = d;
    valid = v;
  endfunction
endclass

// 使用不同的构造函数
Packet p1 = new();              // 8字节数组,valid=0
Packet p2 = new(16);            // 16字节数组,valid=0
Packet p3 = new('{8'hAA, 8'hBB}, 1); // 指定数据,valid=1
```

### 关键要点

- **构造函数名必须为new**
- **不定义时使用默认无参构造函数**
- **可自定义构造函数实现对象初始化**
- **支持构造函数重载(不同参数列表)**
- **构造函数在对象创建时自动执行一次**
- **构造函数中使用this区分成员变量和参数**

### 典型应用

- **配置对象**: 在构造时设置测试参数
- **事务对象**: 创建时初始化地址、数据等字段
- **测试用例**: 构造时配置测试场景
- **验证组件**: 初始化组件ID、配置等

### 最佳实践

1. **总是提供构造函数**: 即使是无参构造函数,也显式定义
2. **初始化所有成员**: 避免使用未初始化的变量
3. **使用参数化构造**: 提高对象创建的灵活性
4. **记录对象创建**: 在构造函数中打印调试信息
5. **使用this关键字**: 区分成员变量和参数,提高可读性
