## 知识点10: 参数化的类

### 核心概念

参数化的类(Parameterized Class)允许在定义类时使用**参数**来指定数据类型、位宽等可配置项。类似于函数的参数让函数更通用,类的参数让类模板化,可以针对不同数据类型或位宽生成不同的类实例。书中强调:参数化类是实现**泛型编程**(Generic Programming)的关键机制,是构建可复用验证组件的基础。

### 参数化类的定义语法

在类名后用`#(参数列表)`声明参数:

```systemverilog
class ClassName #(parameter TYPE = int, parameter int WIDTH = 32);
  // 类体中可以使用TYPE和WIDTH
  TYPE    data;
  bit [WIDTH-1:0] addr;
endclass
```

参数有两种类型:**类型参数**(type parameter)和**值参数**(value parameter)。类型参数用`type`关键字声明,值参数用`parameter`声明。

### 类型参数(type parameter)

类型参数使类的数据类型可配置:

```systemverilog
class Stack #(type T = int);
  T items[$];     // 动态数组,元素类型由参数T决定
  int max_size;

  function new(int max);
    max_size = max;
  endfunction

  function void push(T item);  // push的参数类型也是T
    items.push_back(item);
  endfunction

  function T pop();            // 返回类型也是T
    return items.pop_back();
  endfunction
endclass
```

### 值参数(value parameter)

值参数使类的位宽、大小等数值可配置:

```systemverilog
class AluTransaction #(parameter int ADDR_W = 32, parameter int DATA_W = 32);
  bit [ADDR_W-1:0] addr;
  bit [DATA_W-1:0] data;

  function new(bit [ADDR_W-1:0] a, bit [DATA_W-1:0] d);
    addr = a;
    data = d;
  endfunction

  function void display();
    $display("addr=%0d'd%0d, data=%0d'd%0d", ADDR_W, addr, DATA_W, data);
  endfunction
endclass
```

### 参数化类的实例化

实例化时通过`#(实际参数)`指定参数值:

```systemverilog
// 使用默认参数: T=int, max_size由构造函数传入
Stack s1;                  // 声明句柄(未指定参数时使用默认值)
s1 = new(10);

// 指定类型参数
Stack #(bit [7:0]) byte_stack;   // T=bit[7:0],存储8位数据
Stack #(real)      real_stack;    // T=real,存储浮点数
Stack #(string)    str_stack;     // T=string,存储字符串

// 指定位宽参数
AluTransaction #(16, 16) tr16;    // ADDR_W=16, DATA_W=16
AluTransaction #(64, 128) tr64;   // ADDR_W=64, DATA_W=128
```

### 参数的默认值

定义参数时可以指定默认值,实例化时如果不传参数就使用默认值:

```systemverilog
// 定义时提供默认值
class FIFO #(type T = int, parameter int DEPTH = 16);
  T    buffer[$];
  int  depth;
  function new();
    depth = DEPTH;  // 使用参数值
  endfunction
endclass

// 实例化: 全部使用默认值
FIFO #() fifo_default;       // T=int, DEPTH=16

// 实例化: 只指定部分参数(从左到右)
FIFO #(bit [15:0]) fifo_16;  // T=bit[15:0], DEPTH=16(默认)

// 实例化: 指定所有参数
FIFO #(Transaction, 32) fifo_tr;  // T=Transaction, DEPTH=32
```

### 参数化类与普通类是不同的类型

**重要**: 同一个参数化类,参数不同则类型不同,不能互相赋值:

```systemverilog
Stack #(int)       int_stack;
Stack #(bit [7:0]) byte_stack;

int_stack  = new(10);
byte_stack = new(20);

// int_stack = byte_stack;  // 编译错误! 类型不兼容
```

这就像`int`和`bit [7:0]`是不同类型一样,`Stack #(int)`和`Stack #(bit [7:0])`也是完全不同的类型。

### 在方法中使用参数

类的方法中可以直接使用参数定义的类型和位宽:

```systemverilog
class Register #(parameter int WIDTH = 8);
  bit [WIDTH-1:0] value;

  function void write(bit [WIDTH-1:0] val);
    value = val;
  endfunction

  function bit [WIDTH-1:0] read();
    return value;
  endfunction

  function void display();
    $display("[%0d-bit] value = 0x%0h", WIDTH, value);
  endfunction
endclass

Register #(8)  reg8  = new();
Register #(32) reg32 = new();
Register #(64) reg64 = new();

reg8.write(8'hFF);    // 8位写入
reg32.write(32'hABCD); // 32位写入
```

### 参数化类的继承中传递参数

在子类实例化时,可以将参数传递给父类:

```systemverilog
class MyPacket #(parameter int WIDTH = 32) extends Packet #(WIDTH);
  // MyPacket继承Packet,并将WIDTH参数传递给Packet
  // ...
endclass
```

### 参数化类在验证平台中的典型应用

参数化类在UVM验证平台中广泛使用:

```systemverilog
// 参数化的事务类
class AXITransaction #(parameter int ADDR_W = 32,
                       parameter int DATA_W = 32,
                       parameter int ID_W   = 8);
  bit [ID_W-1:0]   id;
  bit [ADDR_W-1:0] addr;
  bit [DATA_W-1:0] data;
  bit [DATA_W/8-1:0] strb;
  // ...
endclass

// 针对不同总线配置实例化
AXITransaction #(32, 64, 4) axi_tr_32;   // 32位地址,64位数据,4位ID
AXITransaction #(40, 128, 8) axi_tr_40;  // 40位地址,128位数据,8位ID
```

### 关键要点

| 要点 | 说明 |
|------|------|
| **定义语法** | `class ClassName #(参数列表)` |
| **类型参数** | `type T` -- 使数据类型可配置 |
| **值参数** | `parameter int N = 32` -- 使位宽/大小可配置 |
| **默认值** | 参数可设默认值,实例化时可省略 |
| **类型独立性** | 参数不同则类型不同,不可互相赋值 |
| **实例化语法** | `ClassName #(参数) handle = new()` |
| **验证应用** | 可配置Transaction、通用FIFO/Stack、参数化组件 |

### 典型应用场景

- **可配置事务类**: 根据DUT的地址位宽、数据位宽定义不同的事务
- **通用数据结构**: 参数化的Stack、FIFO、Queue等容器类
- **可复用验证IP**: 同一验证组件适配不同配置的DUT
- **总线适配**: AHB/APB/AXI等不同位宽的总线事务建模
