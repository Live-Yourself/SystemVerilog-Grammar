| 特性 | 说明 |
|------|------|
| **书中小节** | 4.4 |
| **核心概念** | 同步驱动、异步采样、时序控制 |

## 6.1 驱动与采样的基本概念

在验证环境中，测试平台需要：
- **驱动(Drive)**：向DUT发送激励信号
- **采样(Sample)**：从DUT接收响应信号

```
┌─────────────────────────────────────────────────────────────┐
│                    驱动与采样示意图                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   测试平台                              DUT                 │
│   ┌─────────┐                         ┌─────────┐         │
│   │         │ ───── 驱动 ──────────► │         │         │
│   │   TB    │                         │   DUT   │         │
│   │         │ ◄──── 采样 ───────────  │         │         │
│   └─────────┘                         └─────────┘         │
│                                                             │
│   驱动: TB → DUT (输入信号)                                 │
│   采样: DUT → TB (输出信号)                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 6.2 同步驱动 vs 异步驱动

### 同步驱动（推荐）

使用时钟块进行同步驱动，确保信号在正确的时机变化。

```systemverilog
// 使用时钟块同步驱动
initial begin
  @(bus.cb);              // 等待时钟同步
  bus.cb.data  <= 8'hA5;  // 非阻塞赋值，在时钟沿后驱动
  bus.cb.valid <= 1'b1;
end
```

### 异步驱动（谨慎使用）

直接驱动接口信号，与时钟无关。

```systemverilog
// 异步驱动 - 不使用时钟块
initial begin
  bus.rst_n = 0;    // 直接赋值，立即生效
  #10;
  bus.rst_n = 1;    // 复位释放
end
```

## 6.3 驱动方式对比

| 方式 | 赋值类型 | 时序 | 适用场景 |
|------|----------|------|----------|
| `bus.cb.sig <= value` | 非阻塞赋值 | 时钟沿后驱动 | 同步信号（推荐） |
| `bus.cb.sig = value` | 阻塞赋值 | 时钟沿后驱动 | 同步信号 |
| `bus.sig = value` | 直接赋值 | 立即生效 | 异步信号（复位等） |
| `bus.sig <= value` | 非阻塞赋值 | 立即生效 | 异步信号 |

## 6.4 采样方式

### 使用时钟块采样（推荐）

```systemverilog
initial begin
  @(bus.cb);  // 等待时钟同步
  
  // 采样DUT输出
  if (bus.cb.ready) begin
    $display("收到响应: data = 0x%02h", bus.cb.data);
  end
end
```

### 直接采样

```systemverilog
// 直接读取信号
always @(posedge bus.clk) begin
  // 此时可能有竞争风险
  sampled_data = bus.data_out;
end
```

## 6.6 双向信号的处理

```systemverilog
interface bi_if;
  logic        clk;
  wire [7:0]   data;    // 双向数据总线
  logic        oe;      // 输出使能
  logic        dir;     // 方向控制
  
  clocking cb @ (posedge clk);
    inout data;         // 双向信号在时钟块中声明为inout
    output oe, dir;
  endclocking
endinterface

// 测试平台驱动双向信号
program bi_driver(bi_if.TB bus);
  initial begin
    // 写操作：TB驱动数据
    bus.cb.dir <= 1;     // 方向：TB→DUT
    bus.cb.oe  <= 1;     // 使能输出
    bus.cb.data <= 8'h55;
    
    // 读操作：DUT驱动数据
    bus.cb.dir <= 0;     // 方向：DUT→TB
    bus.cb.oe  <= 0;
    @(bus.cb);
    // 此时采样 bus.cb.data
  end
endprogram
```

## 6.7 常见驱动采样模式

### 模式1：握手机制

```systemverilog
// 发送数据并等待响应
task send_data(input [7:0] data);
  bus.cb.valid <= 1;
  bus.cb.data  <= data;
  
  // 等待ready响应
  wait(bus.cb.ready);
  
  bus.cb.valid <= 0;
endtask
```

### 模式2：突发传输

```systemverilog
// 连续发送多个数据
task burst_write(input [7:0] data[], int count);
  foreach (data[i]) begin
    @(bus.cb);
    bus.cb.data  <= data[i];
    bus.cb.valid <= 1;
  end
  @(bus.cb);
  bus.cb.valid <= 0;
endtask
```

### 模式3：超时处理

```systemverilog
// 带超时的等待
task wait_ready_with_timeout(input int timeout);
  fork
    begin
      wait(bus.cb.ready);
      $display("收到响应");
    end
    begin
      repeat(timeout) @(bus.cb);
      $display("超时！");
    end
  join_any
  disable fork;
endtask
```

## 6.8 关键要点总结

| 要点 | 说明 |
|------|------|
| **同步驱动** | 使用时钟块和非阻塞赋值 |
| **异步驱动** | 直接赋值，适用于复位等信号 |
| **采样时机** | 在时钟沿之前采样，获取稳定值 |
| **驱动时机** | 在时钟沿之后驱动，避免竞争 |
| **双向信号** | 使用 `inout` 声明，注意方向控制 |
| **避免冲突** | 同一时刻只有一个驱动源 |
