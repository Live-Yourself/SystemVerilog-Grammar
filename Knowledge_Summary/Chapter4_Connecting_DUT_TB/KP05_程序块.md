# 知识点5: 程序块

| 特性 | 说明 |
|------|------|
| **书中小节** | 4.3 |
| **核心关键字** | `program` / `endprogram` |
| **主要作用** | 封装测试平台代码、避免竞争冒险 |

## 5.1 程序块基本概念

**程序块(Program Block)**是SystemVerilog中专门用于封装测试平台代码的构造。它与`module`类似，但具有特殊的时序语义，能够避免测试平台与设计之间的竞争冒险。

### 为什么需要程序块？

```
竞争冒险问题:
┌─────────────────────────────────────────────────────────────┐
│                    module与module的竞争                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   时间轴:                                                   │
│                                                             │
│        时钟沿                                               │
│           │                                                 │
│   ────────┼─────────────────                                │
│           │                                                 │
│           ├── DUT模块执行always块（更新输出）                │
│           └── TB模块执行initial块（采样信号）                │
│                                                             │
│   问题: 谁先执行？不确定！结果可能不一致！                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 5.2 程序块的解决方案

```
┌─────────────────────────────────────────────────────────────┐
│                   program块的执行顺序                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   时间轴:                                                   │
│                                                             │
│        时钟沿                                               │
│           │                                                 │
│   ────────┼─────────────────                                │
│           │                                                 │
│   步骤1:  DUT模块执行always块（更新输出）                    │
│           ↓                                                 │
│   步骤2:  程序块执行initial块（采样信号）                    │
│                                                             │
│   程序块在 Reactive 区域执行                                │
│   → 确保DUT先更新，TB后采样                                 │
│   → 消除竞争冒险！                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 5.3 程序块的基本语法

```systemverilog
program 程序块名 (端口列表);
  // 变量声明
  logic signal1;
  
  // initial块（测试平台的主要逻辑）
  initial begin
    // 测试代码
  end
  
  // 可以包含:
  // - initial 块
  // - function
  // - task
  // - class 定义
  // - assert
  // - covergroup
  
  // 不能包含:
  // - always 块（用 initial forever 替代）
  // - module 实例化
  // - primitive 实例化
  
endprogram
```

## 5.4 program 与 module 的区别

| 特性 | module | program |
|------|--------|---------|
| **执行区域** | Active区域 | Reactive区域 |
| **执行时机** | 与DUT同时执行 | DUT之后执行 |
| **always块** | 可以使用 | **不能使用** |
| **initial块** | 可以使用 | 可以使用（推荐） |
| **实例化module** | 可以 | **不能** |
| **实例化program** | 可以 | 可以 |
| **典型用途** | 设计代码(DUT) | 测试平台代码(TB) |

## 5.5 SystemVerilog仿真调度区域

```
┌─────────────────────────────────────────────────────────────┐
│                    仿真调度区域顺序                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   Active 区域                                               │
│   ├── RTL代码执行 (module的always块)                        │
│   ├── 连续赋值执行                                          │
│   └── DUT逻辑更新                                           │
│           │                                                 │
│           ▼                                                 │
│   Inactive 区域                                             │
│   └── #0 延迟的进程                                         │
│           │                                                 │
│           ▼                                                 │
│   NBA 区域 (Non-blocking Assignment)                        │
│   └── 非阻塞赋值更新                                        │
│           │                                                 │
│           ▼                                                 │
│   Reactive 区域                                             │
│   ├── program块的initial块执行                              │
│   └── 测试平台采样信号                                      │
│           │                                                 │
│           ▼                                                 │
│   Postponed 区域                                            │
│   ├── $monitor, $strobe 执行                                │
│   └── 只读采样                                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘

关键: program块在Reactive区域执行，确保DUT先更新！
```

## 5.6 程序块的使用示例

```systemverilog
// 设计模块(DUT)
module counter(
  input  logic       clk,
  input  logic       rst_n,
  output logic [7:0] count
);
  always_ff @(posedge clk or negedge rst_n) begin
    if (!rst_n)
      count <= 8'h00;
    else
      count <= count + 1;
  end
endmodule

// 程序块(测试平台)
program counter_tb(
  input  logic       clk,
  output logic       rst_n,
  input  logic [7:0] count
);
  initial begin
    // 复位序列
    rst_n = 0;
    repeat(2) @(posedge clk);
    rst_n = 1;
    
    // 监控计数器
    repeat(10) begin
      @(posedge clk);
      $display("[%0t] count = %0d", $time, count);
    end
    
    $finish;
  end
endprogram

// 顶层模块
module top;
  logic       clk;
  logic       rst_n;
  logic [7:0] count;
  
  // 时钟生成
  initial begin
    clk = 0;
    forever #5 clk = ~clk;
  end
  
  // 实例化
  counter     dut (clk, rst_n, count);
  counter_tb  tb  (clk, rst_n, count);
endmodule
```

## 5.7 程序块的注意事项

### 1. 使用 `initial forever` 替代 `always`

```systemverilog
program testbench;
  // ❌ 错误：program中不能使用always
  // always @(posedge clk) ...
  
  // ✓ 正确：使用initial forever
  initial forever begin
    @(posedge clk);
    // 周期性任务
  end
endprogram
```

### 2. 程序块应声明为 automatic

```systemverilog
program automatic testbench;
  // automatic: 每次调用创建新的变量副本
  // 避免静态变量导致的意外共享
endprogram
```

### 3. 程序块自动结束

```systemverilog
program auto_terminate;
  initial begin
    // 执行测试...
  end
  
  // 当所有initial块执行完毕后
  // program块自动调用 $exit
  // 然后执行 final 块（如果有）
  
  final begin
    $display("测试结束，执行清理工作");
  end
endprogram
```

```
│     end                                                     │

│     // 4. 监控/响应                                         │
│     initial forever begin                                   │
│       // 持续监控                                           │
│     end                                                     │
│                                                             │
│     // 5. 清理工作                                          │
│     final begin                                             │
│       // 仿真结束时的清理                                   │

```

## 5.9 关键要点总结

| 要点 | 说明 |
|------|------|
| **关键字** | `program` / `endprogram` |
| **执行区域** | Reactive区域（DUT之后） |
| **避免竞争** | 确保DUT先更新，TB后采样 |
| **禁止always** | 使用 `initial forever` 替代 |
| **自动结束** | 所有initial完成后自动$exit |
| **final块** | 仿真结束前执行清理 |
| **推荐声明** | `program automatic` |
