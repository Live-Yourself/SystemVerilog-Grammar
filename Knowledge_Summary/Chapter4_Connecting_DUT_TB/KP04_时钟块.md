# 知识点4: 时钟块

| 特性 | 说明 |
|------|------|
| **书中小节** | 4.3 |
| **核心关键字** | `clocking` / `endclocking` |
| **主要作用** | 同步信号时序、消除竞争冒险 |

## 4.1 时钟块基本概念

**时钟块(Clocking Block)**是SystemVerilog中用于定义同步信号时序的构造，它明确规定了信号相对于时钟沿的采样和驱动时机，从而消除竞争冒险。

### 为什么需要时钟块？

```
竞争冒险问题:
┌─────────────────────────────────────────────────────────────┐
│                     时钟沿附近的时序                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│        时钟沿                                               │
│           │                                                 │
│     ──────┐     ┌──────                                     │
│           └─────┘                                           │
│           │                                                 │
│           │◄─► 不确定区域                                   │
│           │    信号可能变化                                  │
│           │                                                 │
│     ──────┼─────────────────                                │
│           │                                                 │
│  问题: 测试平台和DUT在同一时刻读写信号                        │
│        → 结果不确定！                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 4.2 时钟块的基本语法

```systemverilog
clocking 时钟块名 @ (时钟事件);
  // 默认时序设置
  default input #1step output #1ns;
  
  // 输入信号：采样时机
  input  signal_in1, signal_in2;
  
  // 输出信号：驱动时机
  output signal_out1, signal_out2;
  
  // 双向信号
  inout  signal_bidir;
endclocking
```

## 4.3 #1step 与 #1ns 的区别（重要）

**step** 是SystemVerilog中的特殊时间单位，代表仿真器的**时间精度**（time precision）。

### 核心区别

| 时间单位 | 含义 | 作用 |
|----------|------|------|
| `#1step` | **一个时间精度步长** | 与仿真时间精度相关，是相对值 |
| `#1ns` | **1纳秒** | 固定的时间值，是绝对值 |

### timescale 指令详解

```systemverilog
`timescale 时间单位 / 时间精度
`timescale 1ns / 100ps
//              │      │
//              │      └── 时间精度：100ps = 1 step
//              │
//              └── 时间单位：1ns
```

### 不同timescale设置下的对比

| 设置 | #1 等于 | #1step 等于 |
|------|---------|-------------|
| `timescale 1ns / 1ps` | 1ns | 1ps |
| `timescale 1ns / 100ps` | 1ns | 100ps |
| `timescale 10ns / 1ns` | 10ns | 1ns |
| `timescale 1us / 1ns` | 1us | 1ns |

### 为什么时钟块默认使用 #1step？

```
┌─────────────────────────────────────────────────────────────┐
│                 时钟块输入采样时机                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│           时钟上升沿                                        │
│               │                                             │
│   ───────────┐     ┌──────────                             │
│               └─────┘                                       │
│               │                                             │
│   ────────────┼─────────────────                            │
│               │                                             │
│        ◄──1step──►                                          │
│        采样点    时钟沿                                      │
│                                                             │
│   使用 #1step 的原因:                                        │
│   1. 在时钟沿之前的"最后一刻"采样                            │
│   2. 获得时钟沿之前的稳定值                                  │
│   3. 不需要知道具体的时间精度                                │
│   4. 自动适应任何 timescale 设置                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 4.4 时钟块的时序关系

```
┌─────────────────────────────────────────────────────────────┐
│                    时钟块时序示意图                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│        时钟上升沿                                           │
│             │                                               │
│   ──────────┐     ┌──────────                              │
│             └─────┘                                          │
│             │                                               │
│    ─────────┼───────────────────                            │
│             │                                               │
│             │◄───►│                                         │
│             │ #1ns│  输入采样偏移                            │
│             │     │  (在时钟沿之前采样)                       │
│             │                                               │
│    ▼────────┼───────────▼──────────                         │
│    采样点   │          时钟沿  驱动点                        │
│             │                  │◄───►│                      │
│             │                  │ #2ns│ 输出驱动偏移          │
│             │                  │     │ (在时钟沿之后驱动)     │
│                                                             │
└─────────────────────────────────────────────────────────────┘

输入采样: 在时钟沿之前采样（获得稳定值）
输出驱动: 在时钟沿之后驱动（避免竞争）
```

## 4.5 输入/输出偏移说明

| 类型 | 默认偏移 | 说明 | 典型值 |
|------|----------|------|--------|
| `input` | `#1step` | 在时钟沿之前采样 | #1ns 或 #1step |
| `output` | `#0` | 在时钟沿之后驱动 | #1ns ~ #10ns |
| `inout` | - | 双向信号 | 根据协议定义 |

## 4.6 时钟块在接口中的使用

```systemverilog
interface bus_if;
  logic clk;
  logic [7:0] data;
  logic       valid;
  logic       ready;
  
  // 定义时钟块
  clocking cb @ (posedge clk);
    default input #1step output #1ns;
    input  ready;          // 在时钟沿前采样
    output data, valid;    // 在时钟沿后驱动
  endclocking
  
  modport TEST (clocking cb, output clk);
endinterface
```

## 4.7 时钟块的使用方式

```systemverilog
// 使用时钟块进行同步操作
module tb(bus_if.TEST bus);
  initial begin
    // 使用时钟块同步驱动（非阻塞赋值）
    bus.cb.data  <= 8'hA5;    // 在时钟沿后驱动
    bus.cb.valid <= 1'b1;
    
    // 使用时钟块同步等待
    @(bus.cb);                // 等待时钟块的一个周期
    wait(bus.cb.ready);       // 等待ready信号
    
    // 循环同步
    repeat(10) @(bus.cb) begin
      // 每个时钟周期执行一次
    end
  end
endmodule
```

## 4.8 时钟块的优点

| 优点 | 说明 |
|------|------|
| **消除竞争** | 明确的采样和驱动时机，避免时序竞争 |
| **同步操作** | 提供统一的同步接口 |
| **可读性好** | 代码清晰表达时序关系 |
| **可维护性** | 时序参数集中管理 |

## 4.9 关键要点总结

| 要点 | 说明 |
|------|------|
| **定义位置** | 通常在interface内部定义 |
| **语法格式** | `clocking 名称 @ (时钟事件) ... endclocking` |
| **输入采样** | 在时钟沿之前采样，获得稳定值 |
| **输出驱动** | 在时钟沿之后驱动，避免竞争 |
| **使用方式** | `接口实例.时钟块名.信号名` |
| **#1step** | 代表时间精度，自动适应timescale |
