# 知识点12: 仿真的结束

| 特性 | 说明 |
|------|------|
| **书中小节** | 4.10 |
| **核心概念** | $finish、$stop、program自动结束、final块 |
| **主要作用** | 正确控制仿真结束时机，确保测试完整执行 |

## 12.1 仿真结束机制概述

SystemVerilog提供了多种控制仿真结束的方式：

```
┌─────────────────────────────────────────────────────────────┐
│                    仿真结束方式对比                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   方式                作用                      推荐度        │
│   ────────────────────────────────────────────────────────  │
│   $finish            结束仿真，返回操作系统        ★★★★★    │
│   $stop              暂停仿真，进入交互模式        ★★☆☆☆    │
│   program自动结束    所有initial完成后自动结束     ★★★★☆    │
│   $exit              结束当前program块            ★★★☆☆    │
│   final块            仿真结束前执行清理           ★★★★★    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 12.2 $finish 与 $stop 的区别

| 特性 | $finish | $stop |
|------|---------|-------|
| **作用** | 终止仿真，返回操作系统 | 暂停仿真，进入交互模式 |
| **后续操作** | 仿真完全结束 | 可在交互模式下继续 |
| **典型用途** | 测试完成时结束 | 调试时暂停检查 |
| **推荐程度** | 正常结束用 | 调试用 |

```systemverilog
module finish_vs_stop;
  initial begin
    // 正常结束仿真
    #100;
    $display("测试完成，结束仿真");
    $finish;        // 推荐：正常结束
    
    // $stop;       // 仅用于调试：暂停仿真
  end
endmodule
```

## 12.3 program块的自动结束机制

**program块具有自动结束特性**：当program中所有initial块执行完毕后，程序块会自动调用`$exit`结束。

```systemverilog
program auto_terminate_tb (
  input  logic clk,
  output logic rst_n
);
  initial begin
    $display("测试开始");
    rst_n = 0;
    repeat(2) @(posedge clk);
    rst_n = 1;
    
    repeat(10) @(posedge clk);
    $display("测试完成");
    
    // 不需要显式调用 $finish
    // program会自动结束
  end
  
  final begin
    $display("仿真结束，执行清理工作");
  end
endprogram
```

## 12.4 final块

**final块**在仿真结束前执行，用于执行清理工作，如生成报告、关闭文件等。

```systemverilog
module final_example;
  int test_count;
  int pass_count;
  int fd;
  
  initial begin
    fd = $fopen("test_report.txt", "w");
    // 执行测试...
    test_count = 100;
    pass_count = 95;
  end
  
  // final块在仿真结束前执行
  final begin
    $display("========== 测试报告 ==========");
    $display("总测试数: %0d", test_count);
    $display("通过数:   %0d", pass_count);
    $display("通过率:   %0.2f%%", real'(pass_count)/test_count*100);
    $display("==============================");
    
    $fclose(fd);
    $display("报告已写入 test_report.txt");
  end
endmodule
```

### final块的注意事项

| 注意点 | 说明 |
|--------|------|
| **执行时机** | 仿真即将结束时（$finish之前） |
| **执行顺序** | 多个final块的执行顺序不确定 |
| **禁止操作** | 不能包含延时（#）、事件等待（@）、fork等 |
| **典型用途** | 生成报告、关闭文件、打印统计信息 |

## 12.5 仿真结束条件

```
┌─────────────────────────────────────────────────────────────┐
│                    仿真结束条件                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   条件1: 显式调用 $finish                                    │
│   条件2: 所有program块执行完毕                               │
│   条件3: 没有任何活动进程                                    │
│   条件4: 执行时间达到预设值（某些仿真器）                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 12.6 常见仿真结束模式

### 模式1：显式$finish（推荐）

```systemverilog
program explicit_finish_tb;
  initial begin
    init_env();      // 初始化
    run_test();      // 运行测试
    wait_all_done(); // 等待所有事务完成
    $finish;         // 显式结束
  end
  
  final begin
    print_report();
  end
endprogram
```

### 模式2：超时保护机制

```systemverilog
module timeout_protection;
  parameter TIMEOUT = 10000;  // 10us超时
  
  initial begin
    fork
      run_test();           // 测试任务
      begin
        #TIMEOUT;
        $error("仿真超时！超过 %0t 未结束", TIMEOUT);
        $finish;
      end
    join_any
    disable fork;
    $finish;
  end
endmodule
```

## 12.7 $finish的参数

| 参数值 | 含义 | 输出内容 |
|--------|------|----------|
| `$finish` 或 `$finish(0)` | 正常结束 | 无额外输出 |
| `$finish(1)` | 带诊断信息 | 打印仿真时间和位置 |
| `$finish(2)` | 详细诊断 | 打印更多信息（内存使用等） |

## 12.8 常见问题与解决方案

### 问题1：仿真不结束

```systemverilog
// 问题：仿真一直运行，不结束
module simulation_hangs;
  initial begin
    #10;
    $display("测试完成");
    // 忘记调用 $finish，且 module 中有 always 块
  end
  
  always @(posedge clk) begin
    // 这个 always 块会一直运行
  end
endmodule

// 解决方案：使用 program 或显式 $finish
```

### 问题2：测试未完成就结束

```systemverilog
// 问题：使用 join_none 导致提前结束
initial begin
  fork
    task1();
    task2();
  join_none    // 错误！立即继续
  // initial 块结束，仿真可能结束
end

// 解决方案：使用 join 或 join_any
initial begin
  fork
    task1();
    task2();
  join_any
  disable fork;
  $finish;
end
```

## 12.9 仿真结束的最佳实践

| 实践 | 说明 |
|------|------|
| **使用program** | 推荐用program封装测试平台，自动结束 |
| **显式$finish** | 在关键位置显式调用$finish |
| **超时保护** | 添加超时机制，防止仿真死循环 |
| **final块清理** | 使用final块生成报告和清理资源 |
| **避免module中的always** | module中的always可能导致仿真不结束 |
| **统一结束点** | 集中管理测试结束逻辑 |

## 12.10 关键要点总结

| 要点 | 说明 |
|------|------|
| **$finish** | 正常结束仿真，返回操作系统 |
| **$stop** | 暂停仿真，仅用于调试 |
| **program自动结束** | 所有initial完成后自动结束 |
| **final块** | 仿真结束前执行，用于清理和报告 |
| **结束条件** | $finish调用、program结束、无活动进程 |
| **超时保护** | 防止仿真无限运行 |
| **推荐方式** | program + final + 显式$finish |
