## 9.1 程序块与模块的基本交互模型

│   │                     │      │                     │     │

## 9.2 执行顺序与时序关系

**核心原则**：program块在 **Reactive区域** 执行，module在 **Active区域** 执行。

```
┌─────────────────────────────────────────────────────────────┐
│                    仿真调度区域执行顺序                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   时钟沿事件触发                                             │
│         │                                                   │
│         ▼                                                   │
│   ┌─────────────────────────────────────────┐              │
│   │  Active 区域                             │              │
│   │  ├── module 的 always 块执行             │              │
│   │  ├── 连续赋值执行                        │              │
│   │  └── DUT 逻辑更新输出                    │              │
│   └─────────────────────────────────────────┘              │
│         │                                                   │
│         ▼                                                   │
│   ┌─────────────────────────────────────────┐              │
│   │  Inactive 区域                          │              │
│   │  └── #0 延迟的进程                      │              │
│   └─────────────────────────────────────────┘              │
│         │                                                   │
│         ▼                                                   │
│   ┌─────────────────────────────────────────┐              │
│   │  NBA 区域 (非阻塞赋值更新)               │              │
│   │  └── <= 赋值生效                        │              │
│   └─────────────────────────────────────────┘              │
│         │                                                   │
│         ▼                                                   │
│   ┌─────────────────────────────────────────┐              │
│   │  Reactive 区域                          │              │
│   │  ├── program 的 initial 块执行 ◄─────── 这里！         │
│   │  └── 测试平台采样DUT输出                 │              │
│   └─────────────────────────────────────────┘              │
│                                                             │
│   关键：program 在 DUT 之后执行，确保采样稳定值！            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 9.3 程序块与模块的通信方式

### 方式1：直接信号连接

```systemverilog
// 模块(DUT)
module counter (
  input  logic       clk,
  input  logic       rst_n,
  output logic [7:0] count
);
  always_ff @(posedge clk or negedge rst_n) begin
    if (!rst_n)  count <= 8'h00;
    else         count <= count + 1;
  end
endmodule

// 程序块(测试平台)
program counter_tb (
  input  logic       clk,
  output logic       rst_n,
  input  logic [7:0] count
);
  initial begin
    rst_n = 0;               // 驱动复位信号
    repeat(2) @(posedge clk);
    rst_n = 1;               // 释放复位
    
    repeat(10) @(posedge clk) begin
      $display("[%0t] count = %0d", $time, count);  // 采样DUT输出
    end
    $finish;
  end
endprogram
```

### 方式2：通过接口连接（推荐）

```systemverilog
// 接口定义
interface counter_if;
  logic        clk;
  logic        rst_n;
  logic [7:0]  count;
  
  clocking cb @ (posedge clk);
    default input #1step output #1ns;
    input  count;
    output rst_n;
  endclocking
  
  modport DUT (input clk, rst_n, output count);
  modport TB  (clocking cb);
endinterface

// 程序块(测试平台) - 使用接口
program counter_tb (counter_if.TB bus);
  initial begin
    bus.cb.rst_n <= 0;
    repeat(2) @(bus.cb);
    bus.cb.rst_n <= 1;
    
    repeat(10) @(bus.cb) begin
      $display("[%0t] count = %0d", $time, bus.cb.count);
    end
    $finish;
  end
endprogram
```

## 9.4 程序块与模块交互的注意事项

### 1. 程序块不能包含 always 块

```systemverilog
program testbench;
  // ❌ 错误：程序块中不能使用always
  // always @(posedge clk) begin ... end
  
  // ✓ 正确：使用 initial forever
  initial forever begin
    @(posedge clk);
    // 周期性任务
  end
endprogram
```

### 2. 程序块不能实例化模块

```systemverilog
program testbench;
  // ❌ 错误：程序块不能实例化模块
  // dut u_dut (...);
endprogram

module top;
  dut u_dut (...);      // ✓ 正确：在模块中实例化
  testbench tb (...);   // 程序块在这里实例化
endmodule
```

### 3. 程序块的自动结束机制

```systemverilog
program auto_terminate_tb;
  initial begin
    // 测试主流程
    run_test();
  end
  
  // 当所有initial块执行完毕后，程序块自动结束
  // 不需要显式调用 $finish
  
  final begin
    $display("测试结束，执行清理工作");
  end
endprogram
```

## 9.5 关键要点总结

| 要点 | 说明 |
|------|------|
| **执行区域** | module在Active区域，program在Reactive区域 |
| **执行顺序** | DUT先执行，测试平台后执行 |
| **通信方式** | 直接连接或通过接口（推荐） |
| **驱动信号** | program可驱动DUT的输入信号 |
| **采样信号** | program采样DUT输出获得稳定值 |
| **禁止always** | program中使用`initial forever`替代 |
| **禁止实例化** | program不能实例化module |
| **自动结束** | 所有initial执行完毕后自动结束 |
