# 知识点1: 将测试平台和设计分开

| 特性 | 说明 |
|------|------|
| **书中小节** | 4.1 |
| **核心目标** | 实现设计与验证的物理和逻辑分离 |
| **关键技术** | 接口(interface)、程序块(program) |

## 1.1 分离原则

SystemVerilog验证方法学的核心原则是将**设计(DUT)**与**测试平台(Testbench)**分开。这种分离带来以下优势：

| 优势 | 说明 |
|------|------|
| **可维护性** | 设计和验证代码独立维护 |
| **可重用性** | 同一测试平台可验证不同设计 |
| **可扩展性** | 易于添加新的测试用例 |
| **团队协作** | 设计与验证工程师可并行工作 |

## 1.2 设计与测试平台的架构

```
┌─────────────────────────────────────────────────────────────┐
│                        顶层模块(Top)                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────┐          ┌─────────────────┐         │
│   │                 │          │                 │         │
│   │   测试平台      │◄────────►│    设计(DUT)    │         │
│   │   (Testbench)   │  接口    │                 │         │
│   │                 │          │                 │         │
│   └─────────────────┘          └─────────────────┘         │
│                                                             │
│   分离原则:                                                  │
│   - 测试平台不知道设计的内部实现                              │
│   - 设计不知道测试平台的存在                                  │
│   - 通过接口进行通信                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 1.3 logic与wire的选择

在连接测试平台和设计时，正确选择信号类型至关重要：

| 类型 | 使用场景 | 特点 |
|------|----------|------|
| **logic** | 单驱动信号 | 可在过程块中赋值，只能有一个驱动源 |
| **wire** | 多驱动信号 | 使用连续赋值，可处理多驱动（线与/线或） |

### 选择决策

```
信号类型选择:

信号有多个驱动源?
│
├─ 是 → 使用 wire
│       (如: 双向总线、多主设备共享信号)
│
└─ 否 → 使用 logic
        (如: 单向控制信号、状态信号)
```

## 1.4 传统Verilog连接方式的问题

在传统Verilog中，测试平台和设计的连接存在以下问题：

| 问题 | 说明 |
|------|------|
| **端口连接繁琐** | 每个信号都需要显式连接 |
| **时序竞争** | 测试平台和设计的时序可能产生竞争条件 |
| **重构困难** | 修改接口需要改动多处代码 |
| **错误率高** | 容易出现端口连接错误 |

## 1.5 SystemVerilog的解决方案

SystemVerilog提供了以下机制来解决这些问题：

| 机制 | 作用 | 对应章节 |
|------|------|----------|
| **接口(interface)** | 封装信号和通信协议 | 4.2 |
| **时钟块(clocking)** | 同步信号时序 | 4.3 |
| **程序块(program)** | 封装测试平台代码 | 4.3 |
| **Modport** | 定义信号方向和视角 | 4.2 |

## 1.6 示例：基本的设计与测试平台分离

### 设计模块(DUT)

```systemverilog
// 简单的DUT示例
module simple_dut(
  input  logic       clk,
  input  logic       rst_n,
  input  logic [7:0] data_in,
  input  logic       valid_in,
  output logic [7:0] data_out,
  output logic       valid_out
);

  always_ff @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
      data_out  <= 8'h00;
      valid_out <= 1'b0;
    end
    else if (valid_in) begin
      data_out  <= data_in;
      valid_out <= 1'b1;
    end
    else begin
      valid_out <= 1'b0;
    end
  end

endmodule
```

### 测试平台(Testbench)

```systemverilog
// 测试平台示例
module simple_tb;
  logic       clk;
  logic       rst_n;
  logic [7:0] data_in;
  logic       valid_in;
  logic [7:0] data_out;
  logic       valid_out;

  // 实例化DUT
  simple_dut dut (
    .clk(clk),
    .rst_n(rst_n),
    .data_in(data_in),
    .valid_in(valid_in),
    .data_out(data_out),
    .valid_out(valid_out)
  );

  // 时钟生成
  initial begin
    clk = 0;
    forever #5 clk = ~clk;
  end

  // 测试激励
  initial begin
    rst_n = 0;
    #10 rst_n = 1;
    
    @(posedge clk);
    data_in = 8'hA5;
    valid_in = 1;
    
    @(posedge clk);
    valid_in = 0;
    
    @(posedge clk);
    $display("data_out = 0x%02h, valid_out = %b", data_out, valid_out);
    
    $finish;
  end

endmodule
```

## 1.7 关键要点总结

| 要点 | 说明 |
|------|------|
| **物理分离** | 测试平台和设计放在不同的模块/程序块中 |
| **逻辑分离** | 使用接口封装通信协议 |
| **类型选择** | 单驱动用logic，多驱动用wire |
| **时序同步** | 使用时钟块避免竞争条件 |
