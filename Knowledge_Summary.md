# SystemVerilog验证学习 - 知识点总结

> 基于《SystemVerilog验证 - 测试平台编写指南》

---

## 第2章 SystemVerilog数据类型

### 知识点1: logic类型

| 特性 | 说明 |
|------|------|
| **类型** | 四值逻辑类型 (0, 1, X, Z) |
| **用途** | 替代Verilog的reg类型 |
| **优势** | 可用于连续赋值和过程赋值 |
| **限制** | 单个变量只能有一个驱动源 |
| **语法** | `logic [width-1:0] var_name;` |

**关键要点:**
- `logic`比`reg`更通用,推荐在SystemVerilog中使用
- 可以被`assign`连续赋值驱动
- 不能有多个驱动(会导致编译错误)

---

### 知识点2: 双状态数据类型

| 类型 | 位宽 | 符号性 | 范围 | 用途 |
|------|------|--------|------|------|
| `bit` | 1位 | 无符号 | 0~1 | 单bit标志 |
| `byte` | 8位 | 有符号 | -128~127 | 字节数据 |
| `shortint` | 16位 | 有符号 | -32768~32767 | 短整数 |
| `int` | 32位 | 有符号 | -2^31~2^31-1 | 常用整数 |
| `longint` | 64位 | 有符号 | -2^63~2^63-1 | 长整数 |

**关键要点:**
- 双状态类型只包含0和1,不包含X和Z
- 仿真性能优于四值类型
- 从四值类型赋值时,X和Z会转换为0
- 验证环境推荐使用双状态类型提高性能

---

### 知识点3: 定宽数组声明

| 数组类型 | 声明语法 | 说明 |
|----------|----------|------|
| 一维数组 | `int arr[8];` | 8个元素,索引0-7 |
| 指定范围 | `int arr[15:8];` | 8个元素,索引15-8 |
| 多维数组 | `int arr[4][8];` | 4行8列 |
| 初始化 | `int arr[4] = '{0,1,2,3};` | 单引号+花括号 |
| 复制初始化 | `int arr[4] = '{4{8}};` | 所有元素为8 |

**关键要点:**
- 定宽数组在编译时确定大小
- 使用`'{}`进行初始化(单引号+花括号)
- 支持`foreach`循环遍历
- 可以使用`%p`格式打印整个数组

---

### 知识点4: 数组操作方法

| 方法 | 功能 | 返回值类型 |
|------|------|-----------|
| `size()` | 返回数组大小 | int |
| `sum()` | 数组求和 | 元素类型 |
| `product()` | 数组求积 | 元素类型 |
| `max()` | 最大值 | 队列 |
| `min()` | 最小值 | 队列 |
| `sort()` | 升序排序 | 无(原地排序) |
| `rsort()` | 降序排序 | 无(原地排序) |
| `reverse()` | 反转数组 | 无(原地排序) |

**关键要点:**
- `max()`和`min()`返回队列类型
- `sort()`/`rsort()`/`reverse()`是原地操作,修改原数组
- 数组可以直接赋值复制: `arr1 = arr2`
- 数组可以直接比较: `if (arr1 == arr2)`

---

### 知识点5: 压缩数组 vs 非压缩数组

| 特性 | 压缩数组 (Packed) | 非压缩数组 (Unpacked) |
|------|------------------|---------------------|
| **声明语法** | `logic [7:0] arr;` | `logic arr[8];` |
| **内存存储** | 连续存储 | 独立存储 |
| **访问方式** | 可整体访问 | 只能逐元素访问 |
| **位操作** | 支持 `arr[3:0]` | 不支持 |
| **典型应用** | 硬件向量、总线 | 寄存器堆、FIFO |
| **混合数组** | `logic [7:0] regfile[16];` | 16个8位向量 |

**语法差异:**
- **压缩数组**: 必须指定范围 `[msb:lsb]`, 不支持简写 `[N]`
- **非压缩数组**: 支持简写 `[N]`, 等价于 `[0:N-1]`

**关键要点:**
- 压缩数组声明: `[size]`在类型**之前**      无简化形式，必须定义上下界
- 非压缩数组声明: `[size]`在变量**之后**    [N] → 简写形式,默认从0开始（升序）,共N个元素   [N] ——> [0:N-1]
- 压缩数组适合硬件建模(总线、寄存器)
- 非压缩数组适合数据结构(FIFO、存储器)
- 混合数组结合两者优点

---

### 知识点6: 数组内存布局

| 特性 | 说明 |
|------|------|
| **维度顺序** | 最右边的维度变化最快 |
| **内存布局** | 类似C语言的行主序 |
| **遍历方式** | `foreach (arr[i, j])` |

**示例:**
```systemverilog
int arr[3][4];  // 3行4列
// 内存顺序: 
// arr[0][0], arr[0][1], arr[0][2], arr[0][3],
// arr[1][0], arr[1][1], arr[1][2], arr[1][3],
// arr[2][0], arr[2][1], arr[2][2], arr[2][3]
```

### 知识点7: 动态数组

| 特性 | 说明 |
|------|------|
| **声明语法** | `data_type name[];` (方括号为空) |
| **分配内存** | `name = new[size];` |
| **调整大小** | `name = new[new_size](name);` 保留原数据 |
| **删除数组** | `name.delete();` |
| **大小查询** | `name.size()` |
| **适用场景** | 运行时才知道大小的数据结构 |

**关键操作:**
```systemverilog
int dyn[];           // 声明
dyn = new[10];       // 分配10个元素
dyn = new[20](dyn);  // 扩展到20,保留原数据
dyn.delete();        // 删除数组
```

**关键要点:**
- 动态数组在运行时分配内存,提高灵活性
- `new[size]`分配新内存,旧数据丢失
- `new[size](old_array)`分配新内存并复制原数据
- 可以直接从定宽数组赋值: `dyn = fixed_arr`
- 仿真开始时才知道数据量大小时使用

**实际应用:**
- 数据包缓冲区
- 临时数据存储
- 仿真配置参数
- 动态生成的测试向量

---

### 知识点8: 关联数组 (Associative Array)

| 特性 | 说明 |
|------|------|
| **声明语法** | `data_type name[*]`; 或 `data_type name[key_type]` |
| **索引类型** | 整数、字符串、类等任意类型 |
| **存储方式** | 稀疏存储,只保存已赋值元素 |
| **内存效率** | 极高,适合超大地址空间 |
| **适用场景** | 稀疏存储器、查找表、配置表 |

**关键方法:**
| 方法 | 功能 | 返回值 |
|------|------|--------|
| `num()` / `size()` | 元素个数 | int |
| `exists(key)` | 键是否存在 | bit |
| `delete(key)` | 删除指定键 | void |
| `first(var)` | 第一个键 | bit |
| `last(var)` | 最后一个键 | bit |
| `next(var)` | 下一个键 | bit |
| `prev(var)` | 前一个键 | bit |

**遍历方式:**
```systemverilog
// 方式1: foreach (推荐)
foreach (arr[key])
  $display("arr[%0d] = %0d", key, arr[key]);

// 方式2: first/next
if (arr.first(key))
  do
    $display("arr[%0d] = %0d", key, arr[key]);
  while (arr.next(key));
```

**关键要点:**
- 关联数组只存储实际写入的元素,节省内存
- 访问不存在的键返回默认值(不会报错)
- `exists()`检查键是否存在
- 支持多种索引类型: `[*]`(任意整数)、`[integer]`、`[string]`
- 内部实现为哈希表,查找效率高

**典型应用:**
- **大容量存储器**: 模拟GB级地址空间,只存储实际访问的地址
- **寄存器文件**: 只定义实际存在的寄存器
- **配置表**: 使用字符串键,提高可读性
- **查找表**: 快速键值查找

**三种数组对比:**
| 数组类型 | 大小确定 | 内存占用 | 适用场景 |
|---------|---------|---------|---------|
| 定宽数组 | 编译时 | 固定 | 固定大小数据 |
| 动态数组 | 运行时 | 连续分配 | 动态大小数据 |
| 关联数组 | 动态增长 | 稀疏存储 | 稀疏数据/大空间 |

### 知识点9: 队列 (Queue)

| 特性 | 说明 |
|------|------|
| **声明语法** | `data_type name[$];` 或 `data_type name[$:max]` |
| **索引范围** | 0 到 `$` (最后一个元素) |
| **内存方式** | 连续存储,高效增长收缩 |
| **适用场景** | FIFO缓冲、消息队列、事务处理 |

**关键方法:**
| 方法 | 功能 | 时间复杂度 |
|------|------|-----------|
| `size()` | 返回队列大小 | O(1) |
| `insert(i, val)` | 在索引i插入 | O(n) |
| `delete(i)` | 删除索引i元素 | O(n) |
| `push_front(val)` | 头部插入 | O(1) |
| `push_back(val)` | 尾部插入 | O(1) |
| `pop_front()` | 头部弹出并返回 | O(1) |
| `pop_back()` | 尾部弹出并返回 | O(1) |

**特殊索引:**
```systemverilog
q[0]    // 第一个元素
q[$]    // 最后一个元素
q[$-1]  // 倒数第二个元素
```

**关键要点:**
- 队列大小可动态调整,无需手动分配
- 头尾操作(push/pop)效率极高 O(1)
- 中间插入/删除效率较低 O(n)
- 支持所有数组方法(sort, min, max等)
- 有界队列可限制最大大小: `q[$:255]`

**典型应用:**
- **FIFO缓冲**: `push_back()`写入, `pop_front()`读取   void'(window.pop_front());显式忽略返回值，即相当于删除队首元素的操作
- **消息队列**: 异步通信的缓冲区
- **事务调度**: 按顺序处理事务
- **数据包缓冲**: 存储变长数据包

**三种容器对比:**
| 特性 | 队列 | 动态数组 | 关联数组 |
|------|------|---------|---------|
| 大小调整 | 自动 | 手动new | 自动 |
| 头尾操作 | O(1)高效 | - | - |
| 索引访问 | O(1) | O(1) | O(log n) |
| 内存占用 | 中 | 低 | 高 |
| 适用场景 | FIFO/队列 | 动态数据 | 稀疏数据 |

### 知识点10: 枚举类型 (Enumeration)

| 特性 | 说明 |
|------|------|
| **声明语法** | `enum [type] {name1=val1, name2=val2} var;` |
| **默认类型** | int类型 |
| **默认值** | 第一个为0,后续递增1 |
| **类型安全** | 只能赋值定义的枚举值 |

**枚举方法:**
| 方法 | 功能 | 返回值 |
|------|------|--------|
| `first()` | 第一个枚举值 | 枚举类型 |
| `last()` | 最后一个枚举值 | 枚举类型 |
| `next(N)` | 后第N个值 | 枚举类型 |
| `prev(N)` | 前第N个值 | 枚举类型 |
| `num()` | 枚举元素个数 | int |
| `name()` | 返回名称字符串 | string |

**枚举声明方式:**
```systemverilog
// 方式1: 自动编号
enum {RED, GREEN, BLUE} color;  // 0, 1, 2

// 方式2: 指定值
enum bit [2:0] {IDLE=0, START=1, STOP=2} state;

// 方式3: 部分指定
enum int {A, B=5, C, D} val;  // A=0, B=5, C=6, D=7

// 方式4: typedef定义
typedef enum bit [3:0] {S0, S1, S2} state_t;
state_t current_state;
```

**关键要点:**
- 枚举提高代码可读性,用名称代替魔术数字
- 枚举具有类型安全,避免无效赋值
- 使用typedef可以创建可复用的枚举类型
- 枚举值可以是任意整数类型(bit, int等)
- name()方法返回枚举值的字符串名称,便于调试

**典型应用:**
- **状态机状态**: FSM状态定义
- **操作码**: 指令集编码
- **配置选项**: 模式选择
- **协议状态**: 通信协议状态机

**枚举与整数转换:**
```systemverilog
enum {A=0, B=1, C=2} val;
int i;

// 枚举 -> 整数 (自动)
i = val;  // 隐式转换

// 整数 -> 枚举 (需要转换)
val = A;  // 直接使用枚举值
// val = 1;  // 警告: 直接赋数字不安全
```

### 知识点11: typedef类型定义

| 特性 | 说明 |
|------|------|
| **作用** | 创建自定义类型别名，提高代码可读性和复用性 |
| **语法** | `typedef existing_type new_type_name;` |
| **优势** | 类型抽象、便于修改、增强可读性 |

**主要应用场景:**
| 类型 | 示例 | 用途 |
|------|------|------|
| 基本类型别名 | `typedef int uint32_t;` | 平台无关类型 |
| 数组类型 | `typedef int int_array[8];` | 简化数组声明 |
| 枚举类型 | `typedef enum {A, B} state_t;` | 状态机定义 |
| 结构体类型 | `typedef struct {int a;} pkt_t;` | 数据包定义 |
| 联合体类型 | `typedef union {int a; byte b[4];} data_t;` | 多视图数据 |

**typedef的优势:**
- **代码可读性**: `state_t` 比 `enum bit[2:0] {...}` 更清晰
- **类型复用**: 可在多处使用同一类型定义
- **易于修改**: 修改类型只需改一处定义
- **类型安全**: 编译器进行类型检查
- **团队协作**: 统一命名规范（常用`_t`后缀）

**命名约定:**
```systemverilog
typedef int    uint32_t;    // _t 后缀表示类型
typedef enum {A, B} state_t;
typedef struct packed {...} packet_t;
```

**关键要点:**
- `typedef`创建新类型，不是变量
- 结构体/枚举/联合体通常配合typedef使用
- 可以定义参数化类型（在类中常用）
- 使用`_t`后缀是SystemVerilog常见命名约定

### 知识点12: 结构体类型 (Struct)

| 特性 | 说明 |
|------|------|
| **作用** | 将多个相关变量组合成一个整体 |
| **成员访问** | 使用点号: `struct_var.member` |
| **类型** | packed(压缩) 和 unpacked(非压缩) |

**packed vs unpacked 对比:**
| 特性 | packed (压缩) | unpacked (非压缩) |
|------|---------------|-------------------|
| **存储方式** | 连续存储 | 独立存储 |
| **整体访问** | 支持 | 不支持 |
| **成员类型** | 仅标量类型(bit/logic) | 任意类型(含string) |
| **位操作** | 支持 `struct[7:0]` | 不支持 |
| **硬件映射** | 直接映射到硬件 | 抽象数据结构 |

**声明语法:**
```systemverilog
// unpacked结构体
typedef struct {
  int    id;
  string name;
} user_t;

// packed结构体
typedef struct packed {
  bit [7:0] data;
  bit       valid;
} packet_t;        // 可整体赋值给 logic [8:0]
```

**结构体操作:**
- **整体赋值**: `struct1 = struct2;`
- **结构体比较**: `if (struct1 == struct2)`
- **%p格式打印**: `$display("%p", struct_var);`
- **嵌套结构体**: 结构体成员可以是另一个结构体
- **结构体数组**: 数组的每个元素是结构体

**关键要点:**
- packed结构体适合硬件建模(寄存器、数据包)
- unpacked结构体适合验证环境(抽象数据)
- 使用`typedef`定义可复用的结构体类型
- packed结构体成员不能是string等复杂类型

### 知识点13: 联合体类型 (Union)

| 特性 | 说明 |
|------|------|
| **作用** | 多个成员共享同一存储空间，实现数据多视图访问 |
| **存储方式** | 所有成员从同一地址开始 |
| **空间占用** | 取最大成员的大小 |

**Union vs Struct 对比:**
| 特性 | Struct | Union |
|------|--------|-------|
| **存储方式** | 成员顺序存储 | 成员共享存储 |
| **空间占用** | 所有成员之和 | 最大成员大小 |
| **同时使用** | 所有成员可用 | 只能用一个 |
| **典型用途** | 数据组合 | 数据视图切换 |

**声明语法:**
```systemverilog
// packed联合体 - 成员位宽必须相同
typedef union packed {
  logic [31:0] word;     // 字视图
  logic [7:0]  bytes[4]; // 字节视图
} word_view_t;

// unpacked联合体 - 成员可不同大小
typedef union {
  int   i_val;
  real  r_val;
} data_t;
```

**典型应用:**
- **寄存器多视图**: 按字节/字/位域访问同一寄存器
- **协议解析**: 同一数据的不同协议头视图
- **大小端转换**: 字节序转换
- **CPU总线**: 数据总线模拟

**使用注意事项:**
- ⚠ 联合体不跟踪当前活动成员
- ⚠ 程序员需自行管理使用哪个成员
- ⚠ packed联合体成员位宽必须相同
- ⚠ 误用成员会导致数据错误

---

## 学习建议

1. **类型选择**:
   - 硬件建模 → `logic`(四值类型)
   - 验证环境 → `int`/`bit`(双状态类型)

2. **数组选择**:
   - 硬件向量 → 压缩数组
   - 数据结构 → 非压缩数组
   - 寄存器堆 → 混合数组

3. **性能优化**:
   - 双状态类型仿真更快
   - 避免X态传播问题
   - 合理使用数组方法简化代码
